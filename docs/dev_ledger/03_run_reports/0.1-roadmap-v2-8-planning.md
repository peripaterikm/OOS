## Key Planning Decisions

- **Replace/amend is the centerpiece.** The v2.7 policy review deferred implementation; v2.8 picks it up as the primary feature.
- **Contract-first.** Item 1.1 formalizes the replace/amend semantics before any code is written, building on the v2.7 policy review's 13 safety requirements.
- **Cleanup before implementation.** Item 1.2 implements parking lot orphan cleanup as a prerequisite for safe replace in item 1.3.
- **Audit trail from day one.** Import history (item 2.1) is implemented immediately after replace/amend so that the first correction is traceable.
- **Parallelizable hardening.** Windows CLI hardening (4.1) and quality gate review (5.1) can run in parallel with the correction workflow items.
- **Deterministic-first preserved.** All correction logic is deterministic; no live LLM/API calls.
- **Advisory-only preserved.** No autonomous portfolio transitions.
- **No new product layers.** v2.8 is correction workflow + operational hardening.
- **Realistic 4–6 week scope.** 9 items, each with focused scope and clear acceptance criteria.
