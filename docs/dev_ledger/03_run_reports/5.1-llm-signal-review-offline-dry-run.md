# Run Report - 5.1 LLM Signal Review Offline Dry-Run

## Summary

Roadmap v2.4 item `5.1` added an offline deterministic LLM Signal Review dry-run. It reads existing discovery artifacts, builds safe review requests, runs deterministic mock review only, validates structured review/JTBD outputs, and writes JSON/Markdown reports.

## Behavior

- Discovery run input expects `artifacts/discovery_runs/<run_id>/candidate_signals.json` and `cleaned_evidence.json`.
- Explicit input paths are also supported.
- Review IDs are deterministic: `llm_review_<review_run_id>_<candidate_signal_id>`.
- Candidate signals are ordered by confidence descending, then signal ID and evidence ID.
- Prompt-safety-blocked items are recorded and do not produce `review_output`.
- Valid mock reviews cite evidence IDs and preserve candidate signal / evidence / source URL traceability.
