## Summary

Implemented a deterministic, fail-closed founder decision import layer. The importer reads explicit founder decisions (JSON array or JSONL), validates against the founder inbox v2 index, converts valid decisions into `FounderDecisionV2`, `FounderFeedbackMapping`, `FounderPreferenceProfile`, and `ParkingLotRecord` artifacts, and updates the weekly run manifest.

## Design Verification

| Check | Status |
|-------|--------|
| Import only explicit founder decisions? | ✅ No auto-generation or inference |
| Avoid autonomous decisions? | ✅ `advisory_only: true` throughout |
| Avoid portfolio mutation? | ✅ No portfolio state changes |
| Avoid status/dashboard scope? | ✅ Scoped to import artifacts only |
| Avoid live APIs/LLMs? | ✅ `no_live_api: true`, `no_live_llm: true` |
| Fail-closed (no partial writes)? | ✅ Any invalid input → 0 artifacts written |
| Idempotent? | ✅ Re-import detected and rejected |
| Tests avoid real artifacts/? | ✅ All tests use temp directories |
| Deterministic? | ✅ No randomness, no external calls |

## Known Limitations

1. `weekly_opportunity_review.json` is not rebuilt after import — requires full pipeline re-run via `run-weekly-cycle-v2`.
2. `next_best_actions.json` is not rebuilt — same rationale.
3. `source_urls` use a deterministic placeholder (`urn:oos:founder_import:placeholder`) when inbox items don't carry source URLs; feedback mapping validation requires at least one URL.
4. Decision import does not trigger revisit matching — that happens during the next `build_weekly_cycle()` run.
