## Risk Analysis Summary

| Risk | Verdict |
|------|---------|
| Partial undo writes | Mitigated by all-or-nothing write policy |
| Stale source URLs after undo | Mitigated by source URL traceability check (U-R8) |
| Orphaned parking lot records | Mitigated by same cleanup logic as replace mode |
| Amend undo creates no inconsistency | Verified — amend does not rebuild derived artifacts |
| Multi-step undo ambiguity | Out of scope — only U1 (undo-last) considered safe |
| Archive corruption | Mitigated by fail-closed + archive integrity verification (U-R12) |
| Accidental undo | Mitigated by explicit CLI flag (U-R1) |
