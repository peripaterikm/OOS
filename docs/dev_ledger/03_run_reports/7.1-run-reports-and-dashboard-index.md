# Run Report 7.1: Run Reports and Dashboard Index

**Date**: 2026-05-07
**Status**: Complete
**Tests**: 44 focused / 1357 total / 0 failures

## Implementation Summary

Added deterministic run reports and a cross-run dashboard index for v2.6 weekly runs. The implementation builds on `WeeklyRunManifest` and `WeeklyCycleStatus` as sources of truth, aggregates existing artifacts, and writes per-run report artifacts and a cross-run dashboard.

### Key deliverables

1. **`src/oos/weekly_run_reports.py`** — Core module with:
   - `WeeklyRunReport` model (per-run aggregation)
   - `WeeklyDashboardIndex` model (cross-run aggregation)
   - `WeeklyDashboardRunSummary` model (per-run dashboard row)
   - `build_weekly_run_report()` — builds report from run directory
   - `write_weekly_run_report()` — writes `run_report.json` + `run_report.md`
   - `render_weekly_run_report_markdown()` — human-readable Markdown
   - `build_weekly_dashboard_index()` — scans all runs
   - `write_weekly_dashboard_index()` — writes `dashboard_index.json` + `dashboard.md`
   - `render_weekly_dashboard_markdown()` — human-readable Markdown with summary table

2. **`src/oos/cli.py`** — Two new CLI subcommands:
   - `build-weekly-run-report-v2 --project-root <dir> --run-id <id>`
   - `weekly-dashboard-v2 --project-root <dir> [--json]`

3. **`src/oos/weekly_cycle_builder.py`** — Replaced placeholder run report logic with real `build_weekly_run_report()` / `write_weekly_run_report()`.

### Integration points

- Uses `build_weekly_cycle_status()` as source of truth for per-run health
- Reads manifest from disk (after it's written in the builder)
- Dashboard scans `artifacts/weekly_runs/` for all run directories
- Stale `run_report.json`/`run_report.md` are cleaned before rebuild to preserve determinism

### Safety

- No live APIs, no live LLMs
- No portfolio mutations
- No decision import
- No weekly cycle rebuild
- No change to founder inbox or status command semantics
- All tests use temp directories only
