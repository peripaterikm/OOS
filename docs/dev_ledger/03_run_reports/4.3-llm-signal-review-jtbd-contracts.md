# Run Report - 4.3 LLM Signal Review and JTBD Contracts

## Summary

Roadmap v2.4 item `4.3` added contract-only support for future LLM-assisted signal review and JTBD extraction. The implementation is deterministic, evidence-bound, prompt-safety aware, and disabled from live execution by default.

## Changes Made

- Added `src/oos/llm_signal_review.py` with signal review input/output contracts, JTBD statement contracts, prompt builders, safe request construction, structured JSON parsing, validation, and deterministic mock review.
- Updated `src/oos/llm_contracts.py` so the local-preview budget policy allows the `llm_signal_review` task type.
- Added `tests/test_llm_signal_review_contracts.py` covering prompt content, prompt safety, JSON parsing, validation failures, deterministic mock behavior, and no provider calls.
- Updated `tests/test_llm_contracts_and_budgets.py` for the local-preview task type allowance.
- Updated the Roadmap v2.4 state to current item `5.1`, completed `9 / 17`, remaining `8 / 17`.
- Updated the Dev Ledger project state and roadmap checkpoint compatibility test.

## Contract Details

- Signal review prompts require evidence-bound review only, no invented facts, evidence ID citations, asymmetric weak-signal prior, marketing/generic-copy discrimination, and structured JSON output.
- JTBD output captures actor, situation, desired outcome, when/want-to/so-that framing, current workaround, cited evidence IDs, and confidence.
- Output validation is fail-closed for missing/unknown citations and invalid contract fields.

## Safety Details

- `build_safe_signal_review_request` applies `prompt_safety` before returning an `LLMRequest`.
- Secrets, private keys, and card-like sensitive content remain blocked by the existing prompt-safety policy.
- No provider `.complete()` call is made by the review contract layer.
