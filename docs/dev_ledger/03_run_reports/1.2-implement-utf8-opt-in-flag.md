## Key Design Decisions

1. **Centralized `src/oos/output_modes.py`** — Single source of truth for symbol mappings. Both renderers import from this module.
2. **Symbol mapping dict** — Keys: `success`, `failure`, `warning`, `none`, `arrow`, `dash`, `corrected`. Both modes return the same key set; only values differ.
3. **Fail-closed validation** — Unknown mode raises `ValueError` with clear message listing valid modes.
4. **No automatic detection** — No `sys.stdout.encoding`, `chcp`, or `GetConsoleOutputCP()` calls. Explicit `--utf8` flag only.
5. **ASCII-safe default unchanged** — All default output passes `ord(c) < 128` check.
6. **JSON artifacts mode-independent** — `weekly_cycle_status_to_json()`, `run_report.json`, `dashboard_index.json` produce identical output regardless of mode.

## Audit: import-founder-decisions-v2

Terminal output prints only labels, values, and booleans. No status symbols. `--utf8` not needed. Confirmed by test that `--help` does not list `--utf8`.
