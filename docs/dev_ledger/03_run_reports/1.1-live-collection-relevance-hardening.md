## Reason For Hardening

Manual live smoke runs showed technically working collectors but noisy results:

- HN surfaced broad generic small-business content.
- GitHub surfaced marketing/product-pitch and generated-content issues.
- RSS attempted to fetch search query text as a URL.
- Summaries contained HTML entities/tags such as `doesn&#x27;t` and `<p>`.
- GitHub summaries showed mojibake fragments such as `рџ`, `вЂ`, and `рџ“`.
- Candidate confidence was too flat across unrelated signals.

## Changes Made

- Added deterministic HTML/entity cleanup in the evidence cleaner.
- Added explicit UTF-8 decode with replacement in live collector fetch paths.
- Added `ai_cfo_smb` relevance scoring and low-relevance review/default behavior.
- Added marketing/generated-content downgrade rules.
- Added more discriminative CandidateSignal confidence scoring.
- Updated `ai_cfo_smb` query templates to finance-specific HN/GitHub-friendly terms.
- Added RSS URL validation and nonfatal `rss_feed_url_missing` skip behavior.
- Added tests for observed HN/GitHub/RSS live-quality failures.
