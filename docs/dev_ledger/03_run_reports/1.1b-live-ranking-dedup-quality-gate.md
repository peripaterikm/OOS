## Reason for Patch

Manual live smoke after item 1.1 showed improved finance relevance, HTML cleanup, scoring spread, and RSS skip behavior, but also revealed remaining quality issues:

- HN top package could duplicate the same source URL and pain summary.
- GitHub install/tutorial/generic finance copy could rank above actual user pain.
- GitHub summaries still showed common mojibake fragments such as `todayвЂ™s` and `optionalвЂ”they`.
- Obvious marketing/install/tutorial junk did not reliably contribute to `noise_count`.
- Real user pain such as separate spreadsheet workarounds, balance sheet requests, and invoice/payment spreadsheet pain needed to outrank generic content.

## Changes Made

- Added founder-package top signal deduplication by `source_type + source_url`, with deterministic fallbacks and tie-breaking.
- Added common mojibake repair in the cleanup layer and removed broken emoji fragments from summaries.
- Strengthened GitHub quality gates for install/tutorial/SEO/service-page/generic finance copy.
- Preserved genuine user pain markers such as `Describe the problem`, `I would like`, `we would need`, `manual spreadsheet`, and balance sheet requests.
- Added explicit noise coverage for obvious GitHub junk while preserving ambiguous HN/GitHub content as low-confidence review.
- Improved ranking tests so real user pain outranks generic install/marketing/tutorial content.
