## Key Behaviors

### Replace Mode
- Requires explicit `--replace-review-items` flag
- Only replaces listed `review_item_id` values
- Rejects incoming items not in the replace list
- Rejects non-existing review_item_ids
- Full derived artifact rebuild (feedback mappings, preference profile, parking lot)
- Orphaned parking lot records cleaned
- New parking lot records created for PARK/REVISIT
- Real source URLs preserved; `urn:oos:*` rejected
- `import_history.json` written with correction entry
- All-or-nothing: no partial writes on failure
- Deterministic: same input yields same output

### Amend Mode
- Requires explicit `--amend-notes-only` flag
- Updates notes and reason categories only
- Rejects decision value changes
- Does NOT rebuild feedback mappings, preference profile, or parking lot
- Preserves source URLs
- `import_history.json` written with correction entry

### Default Behavior (unchanged)
- Re-import without flags still fails closed
- Duplicate/unknown/invalid review_item_ids still rejected
- All v2.6/v2.7 safety properties preserved

## Risks / Follow-up

- No replace-all mode (deferred to v2.9+ per contract)
- No undo/rollback (v2.9+ candidate)
- Run report/dashboard must be regenerated manually via existing CLI commands
