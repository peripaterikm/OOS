# Run Report 6.3 - Parking Lot / Revisit Logic

## Branch

`feat/v2-5-block-5-founder-feedback-loop`

## Summary

Implemented a deterministic parking lot / revisit logic layer that keeps parked opportunities visible and matches new evidence against them using simple, inspectable rules. Uses explicit pattern keys first, then falls back to conservative normalized token overlap, then substring matching. All matching is deterministic — no embeddings, ML, LLM, network calls, or live APIs. The founder remains the final decision-maker; no autonomous portfolio transitions occur. Revisit matches are surfaced in the weekly opportunity review package's revisit_queue section.
