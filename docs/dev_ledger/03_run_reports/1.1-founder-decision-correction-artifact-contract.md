### Parking lot integrity policy

- Orphaned records (those whose `source_decision_id` no longer exists in `founder_decisions_v2.json`) are **removed** from active `parking_lot_records.json`.
- Supersession is tracked in `import_history.json`, not in parking lot records.
- Deletion is allowed for orphaned records. Superseding is preferred over silent deletion.
- Active records are never deleted unless their source decision is explicitly replaced.
- Old parking lot state is preserved in `replaced_decisions/` archive.

### Import history / audit trail

- `CorrectionEntry` model: 12 fields including `correction_id`, `corrected_at`, `correction_mode`, `replaced_review_item_ids`, `old_decision_ids`, `new_decision_ids`, `old_artifact_checksums`, `new_artifact_checksums`, `warnings`, `errors`, `advisory_only`, `no_live_api`, `no_live_llm`.
- `ImportHistoryLog` model: `schema_version`, `run_id`, `entries`.
- File location: `{run_dir}/import_history.json`.
- Append-only; entries never modified or deleted.

### Source URL traceability guarantee

- Zero `urn:oos:*` placeholder URNs after any correction.
- Real `http`/`https` source URLs from inbox index must survive correction.
- `check_source_url_traceability()` must pass after correction.
- Missing source URLs fail closed unless explicitly exempt.
