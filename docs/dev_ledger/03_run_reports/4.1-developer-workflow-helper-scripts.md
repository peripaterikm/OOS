## What was done
Created four Windows-native PowerShell developer workflow helper scripts and a focused test suite (34 tests).

### Scripts created

1. **`scripts/dev-snapshot.ps1`** — Creates a local diagnostic snapshot file under `_local_hold/dev_snapshots/`. Captures timestamp, branch, git status, git log, git diff --check, HEAD summary, and roadmap state excerpts. No destructive operations.

2. **`scripts/dev-validate-final.ps1`** — Runs final local validation before commit/PR/merge. Executes git status, git diff --check, git diff --check HEAD~1..HEAD, full unittest discovery (or focused test via `-FocusedTest`), and `scripts/oos-validate.ps1`. Prints PASS/FAIL summary. Supports `-SkipFullTests`, `-SkipOOSValidate`, and `-FocusedTest` parameters. Returns non-zero on failure.

3. **`scripts/dev-pr-readiness.ps1`** — Read-only PR readiness check. Prints branch, HEAD commit, git status, comparison to origin/main, git diff --check, secure file check (no .env, .venv, artifacts, reports in staging), roadmap/dev ledger update status, and mini-epic/run report existence. Prints BLOCKER/WARNING summary. Returns non-zero if blockers exist.

4. **`scripts/dev-post-merge-sync.ps1`** — Print and optionally perform safe local sync after PR merge. Default behavior is dry-run/instructions-only. With `-ExecuteSafeSync`, runs: `git switch main`, `git pull --ff-only`, `git fetch --prune`, `git status --short`. Never deletes branches, pushes, merges, or tags.

### Safety guarantees verified

- **No forbidden commands**: None of the 4 scripts contain `git add`, `git commit`, `git push`, `git merge`, `git tag`, `git reset`, `git clean`, `gh pr create`, `gh pr merge`, `Remove-Item`, `rm`, `del`.
- **No live API/LLM calls**: No script uses `Invoke-RestMethod`, `Invoke-WebRequest`, `curl`, `wget`, or API URLs.
- **Windows-native only**: No bash, no shebang, no WSL.
- **Constrained writes**: `dev-snapshot.ps1` writes only under `_local_hold/dev_snapshots/` (verified via temp dir execution test).
- **Dry-run default**: `dev-post-merge-sync.ps1` defaults to instructions-only; execution mode requires explicit `-ExecuteSafeSync`.
- **No branch deletion**: `dev-post-merge-sync.ps1` explicitly states no branches are deleted.
- **No PR automation**: `dev-pr-readiness.ps1` explicitly states PR creation must be manual.
