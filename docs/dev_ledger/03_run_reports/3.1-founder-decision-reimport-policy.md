## Policy Review Summary

### Current behavior analyzed
- `validate_founder_decision_inputs()` — duplicate `review_item_id` rejection at line 300-311
- `_merge_decisions()` — replaces by `opportunity_id` at lines 720-730
- `_merge_parking_lot_records()` — deduplicates by `record_id` but never removes orphaned records
- Source URL traceability contract satisfied (items 1.1–2.1)
- All downstream artifacts (feedback mappings, preference profile, parking lot) are touched during import

### Critical finding
**Parking lot record cleanup is the blocking concern.** The current `_merge_parking_lot_records()` only adds/updates records. A PARK→PROMOTE replacement would leave an orphaned `ParkingLotRecord` with a `source_decision_id` pointing to a deleted decision. Fixing this requires:

1. A new `_remove_orphaned_parking_lot_records()` function.
2. Archival logic for replaced decisions (`replaced_decisions/` subdirectory).
3. Manifest tracking of replaced decision IDs.
4. At least 15 focused tests.

This is non-trivial and crosses the v2.7 "hardening, not feature expansion" boundary.
