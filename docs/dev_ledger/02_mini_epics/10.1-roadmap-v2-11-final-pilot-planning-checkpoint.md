Compatible with existing `RawEvidence` model.
3. **Source Registry and Allowlist Policy** — [`config/source_registry.json`](../../../config/source_registry.json) listing HN and GitHub Issues with status/metadata; [`docs/contracts/source_allowlist_policy.md`](../../contracts/source_allowlist_policy.md) defining acceptance (stable URL, deterministic access, fixture availability, ToS/robots compliance), suspension, removal criteria, and approval process.
4. **Hacker News Connector Hardening Plan** — [`docs/decisions/hacker_news_connector_hardening_plan.md`](../../decisions/hacker_news_connector_hardening_plan.md) covering current assessment, preferred API method (Firebase + Algolia), coverage gaps (comments, Ask HN, Show HN, Launch HN), source_url traceability audit, noise filter recommendations, rate-limit compliance, fixture quality assessment, and prioritized implementation changes. Implementation-ready.
5. **GitHub Issues Connector Hardening Plan** — [`docs/decisions/github_issues_connector_hardening_plan.md`](../../decisions/github_issues_connector_hardening_plan.md) covering current assessment, PR filtering effectiveness, repo allowlist coverage, keyword search quality, label/comment/state capture, source_url traceability audit (including removal of `github://` fallback), rate-limit/auth policy, fixture quality assessment, and prioritized implementation changes. Implementation-ready.
6. **Product Hunt Feasibility Plan** — [`docs/decisions/product_hunt_feasibility_connector_plan.md`](../../decisions/product_hunt_feasibility_connector_plan.md) covering API assessment (GraphQL), data model mapping, source_url traceability, anti-hype scoring design, fixture strategy, access method, known limitations, and implementation scope estimate. Explicitly states: no implementation until API feasibility and auth requirements are documented and founder-approved.

### Operational Pilot Design (Items 7–9)

7. **Operational Discovery Pilot Reorientation** — [`docs/decisions/operational_discovery_pilot_reorientation_v2_11.md`](../../decisions/operational_discovery_pilot_reorientation_v2_11.md). Strategic decision to re-scope v2.11 from broad source expansion into a bounded operational pilot on HN + GitHub Issues (with Stack Exchange as optional stretch). All other sources deferred to v2.14+ conditional on pilot Go decision in v2.13.
8. **PainCluster Contract and Scoring Formula** — [`docs/contracts/pain_cluster_contract.md`](../../contracts/pain_cluster_contract.md). First-class artifact for cross-source pain consolidation. Defines 19 fields (cluster_id through notes), cross-source consolidation rule (same pain across HN + GitHub Issues + Stack Exchange → one PainCluster), and explicit pain-first scoring formula: `overall = 0.25*pain_explicitness + 0.20*recurrence + 0.15*business_cost + 0.15*icp_fit + 0.10*source_reliability + 0.10*freshness + 0.05*actionability - 0.20*noise_risk`. All 8 components defined with deterministic 0.0/0.5/1.0 scoring guidance. No LLM required.
9. **Pilot Run Design and Source Quality Report Contract** — [`docs/contracts/operational_discovery_pilot_run_contract.md`](../../contracts/operational_discovery_pilot_run_contract.md). Defines 14-phase pilot run lifecycle (preflight → collection → validation → extraction → classification → dedup → clustering → scoring → opportunity framing → source quality reporting → founder review package → weekly report → feedback ingestion → retrospective), 8-section source quality report, 18 source quality metrics, founder review loop (PROMOTE/PARK/KILL/NEEDS_MORE_EVIDENCE/REVISIT_LATER with feedback into scoring), 10-section weekly pilot report, success/failure criteria, Go/No-Go criteria, validation policy, source-specific rules, traceability requirements, operational constraints, and retrospective questions.
