# 4.2 GitHub Issues Collector

## Goal

Implement the second real-source collector adapter for public GitHub Issues while preserving fixture-first, offline-safe Roadmap v2.3 source intelligence rules.

## Scope

- Add `GitHubIssuesCollector` conforming to the `BaseCollector` contract.
- Add deterministic GitHub issue-to-`RawEvidence` mapping.
- Add fixture-first collection behavior with live network disabled by default.
- Add a standard-library live path gated by both collector config and scheduled item `live_network_enabled`.
- Add focused tests for source compatibility, mapping, traceability, privacy, max results, malformed issues, pull request filtering, and the Source Registry -> Query Planner -> Scheduler -> GitHub collector fixture flow.

- No signal extraction.
- No weekly discovery CLI integration.
- No internet/API calls during tests or validation.
- No live LLM calls.

## GitHub Collector Behavior

- Supports only `source_id=github_issues` and `source_type=github_issues` by default.
- Rejects unsupported scheduled collection items.
- Uses fixture payloads when provided.
- Returns empty evidence safely when live networking is disabled and no fixture payload is provided.
- Supports GitHub issue search for the optional future live path using only the Python standard library.

## Fixture-First Behavior

- Focused tests use in-memory GitHub issue fixture payloads.
- Empty fixture responses return an empty evidence list.
- Malformed or incomplete issues without an issue identity are skipped.
- Recoverable issues use deterministic title/body/source URL fallbacks.

## Live-Network-Disabled-By-Default Behavior

- `allow_live_network` defaults to `False`.
- The collector requires both `allow_live_network=True` and `scheduled_item.live_network_enabled=True` before using the standard-library live path.
- Tests assert no live fetch is attempted when live networking is disabled.
- No internet/API calls were made during validation.

## RawEvidence Mapping

- `evidence_id`: deterministic `raw_github_issue_<issue_id>`.
- `source_url`: issue `html_url` when available.
- `title`: issue `title`, then `GitHub issue <issue_id>`.
- `body`: issue `body`, then title fallback.
- `source_id`, `source_type`, `topic_id`, and `query_kind` are preserved from the scheduled item.
- `content_hash` uses existing deterministic `RawEvidence` title/body hashing.
- `raw_metadata` preserves safe issue metadata such as issue ID, node ID, issue number, repository URL, comments URL, label names, state, timestamps, comment count, safe reaction summary, pull request presence, user presence, query plan ID, and dedup key.

## Privacy / Author Context

- `author_or_context` is always role/context-only: `unverified public issue reporter`.
- Raw GitHub usernames/logins are not stored in `author_or_context`.
- Raw GitHub usernames/logins are not stored in `raw_metadata`; only `user_present` is retained.

## Pull Request Filtering

- Pull request-shaped issue records are skipped by default.
- This keeps v2.3 item 4.2 focused on user pain/workaround/feature request evidence from issue-level records.
