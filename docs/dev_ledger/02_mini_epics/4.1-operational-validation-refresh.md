## Implementation decisions

1. **Separate v2.9 module** — Created `src/oos/v2_9_operational_validation.py` rather than modifying v2.8 module. This keeps v2.8 stable and makes v2.9-scoped validation self-contained.
2. **Symbol contract check only** — ASCII/UTF-8 checks verify the output_modes.py contract directly (static symbol inspection) rather than testing CLI output. CLI output verification (steps V4–V6) runs actual weekly cycle builder for end-to-end confirmation.
3. **Smoke script minimal change** — Only Step 8 changed; added `missing_count` and `validation_passed` checks. No destructive cleanup or git commands added.
4. **Temp-root only** — All validation uses `TemporaryDirectory`. No real `artifacts/` written.
5. **No business logic changes** — This is pure validation/verification; no correction semantics, source URL scanner, or pipeline changes.
