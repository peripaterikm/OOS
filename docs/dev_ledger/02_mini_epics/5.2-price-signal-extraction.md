# Mini-Epic 5.2 - Price Signal Extraction

## Roadmap Item

- Roadmap: `docs/roadmaps/OOS_roadmap_v2_4_signal_quality_and_ai_layers_checklist.md`
- Item: `5.2 Price signal extraction`
- Branch: `feat/5-x-llm-review-offline-dry-run`
- Commit scope: local only; no push/PR/merge/tag/release.

## Goal

Extract explicit budget, spend, effort-cost, price complaint, and willingness-to-pay hints from evidence without inventing budgets or normalizing values.

## In Scope

- Add serializable `PriceSignal` model and artifact kind.
- Add deterministic regex/rule baseline in `src/oos/price_signal_extractor.py`.
- Extract `current_spend_hint`, `effort_cost_hint`, `price_complaint`, `willingness_to_pay_indicator`, `evidence_cited`, and `confidence`.
- Add future-only LLM prompt contract requiring exact evidence citation and no invented budgets.
- Add a narrow scoring boost only when explicit price evidence exists.
- Add founder discovery package price hint display and discovery-run `price_signals.json`.
- Add focused tests for extraction, no-invention behavior, scoring boost, prompt contract, artifact roundtrip, and founder package display.

## Out Of Scope

- Live LLM/provider execution.
- Internet/API calls.
- Price normalization or monthly value estimation.
- New dependencies.
- Broad scoring redesign.
- Full Founder Package Quality Upgrade item `7.2`.

## Safety

- Deterministic extraction only.
- Evidence snippets are required for non-null price signal fields.
- Vague text such as budget visibility does not create a price signal.
- Scoring boost is small, explicit, and cannot bypass existing noise/human-review caps.

## Acceptance

- Focused price signal tests pass.
- Full test discovery passes.
- Standard validation script passes.
- Roadmap and project state are updated to the next planned item.
