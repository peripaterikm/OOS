# 3.1 Semantic Clustering

## Goal

Group canonical signals by meaning instead of repeated words.

## Context

Clustering must respect deduplication so duplicate raw recurrence does not inflate cluster strength.

## Inputs

- Canonical signals
- AI metadata contract
- Optional signal meaning context
- Stubbed clustering provider payload

## Outputs

- `src/oos/semantic_clustering.py`
- Structured semantic cluster records
- Low-confidence fallback rule
- Focused and acceptance tests

## Key Decisions

- Use `canonical_signal_set()` by default.
- If all clusters are below confidence `0.4`, fallback to simple per-signal grouping.
- Keep clustering standalone; do not wire into `run-signal-batch` yet.

## Alternatives Rejected

- Live LLM/API calls.
- Using duplicate raw signals as cluster recurrence input.

## Validation

- Semantic clustering tests.
- Acceptance guards for no live calls and no premature orchestrator wiring.
- Full unittest discovery and `scripts/verify.ps1`.

## Known Limitations

- Provider responses are static/stubbed.

## Next Step

Detect contradictions and merge candidates before opportunity framing.
