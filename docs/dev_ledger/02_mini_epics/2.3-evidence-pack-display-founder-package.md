# 2.3 Evidence Pack Display In Founder Package

## Goal

Make founder-facing discovery packages show compact, traceable EvidencePack sections so each cluster/opportunity seed can be reviewed from grounded evidence rather than loose signal lists.

## Scope

- Added deterministic EvidencePack display helpers for founder package JSON and Markdown outputs.
- Included compact evidence-pack fields: pack ID, cluster ID, source signal/evidence counts, source diversity, recurrence count, evidence IDs, source signal IDs, source URLs, top summaries, linked price signal IDs, weak pattern IDs, kill warning IDs, and risk notes.
- Wired discovery weekly founder packages to include the in-memory EvidencePack objects already generated by the 2.2 builder.
- Preserved empty-state behavior when no evidence packs exist.

## Out Of Scope

- No opportunity generation.
- No LLM synthesis.
- No founder decision automation.
- No new collectors or live collection.
- No source expansion or architecture redesign.

## Acceptance Criteria

- [x] Founder package JSON includes an `evidence_packs` section when packs exist.
- [x] Founder package Markdown includes a readable Evidence packs section.
- [x] Empty evidence-pack state renders without crashing.
- [x] Evidence IDs, source signal IDs, and source URLs remain traceable.
- [x] Linked price, weak-pattern, kill-warning, and risk-note metadata remains compact.
- [x] Existing founder package quality sections still render.
- [x] Fixture-only validation passes without live internet/API or LLM calls.
