## Goal
Add Windows-native PowerShell helper scripts for common developer operations: git snapshot, final validation check, post-merge sync, and PR readiness check. One command per operation, no magic.

## Scope
- Create `scripts/dev-snapshot.ps1` — captures a local diagnostic snapshot.
- Create `scripts/dev-validate-final.ps1` — runs final validation before commit/PR/merge.
- Create `scripts/dev-pr-readiness.ps1` — read-only PR readiness checklist.
- Create `scripts/dev-post-merge-sync.ps1` — safe post-merge housekeeping (default dry-run).
- Create `tests/test_dev_workflow_scripts.py` — focused tests (34 tests).

## Key Design Decisions

1. **PowerShell-only, Windows-native.** No bash/zsh/WSL constructs.
2. **Read-only default.** All scripts default to read-only inspection. The only exception is `dev-post-merge-sync.ps1 -ExecuteSafeSync`, which runs only safe git operations (switch main, pull --ff-only, fetch --prune, status).
3. **No forbidden commands.** No script contains `git add`, `git commit`, `git push`, `git merge`, `git tag`, `git reset`, `git clean`, `gh pr create`, `gh pr merge`, `Remove-Item`, `rm`, `del`.
4. **Safe error handling.** All scripts use `Set-StrictMode -Version Latest; $ErrorActionPreference = 'Stop'`.
5. **Comment-based help.** All scripts have `.SYNOPSIS`, `.DESCRIPTION`, `.PARAMETER`, `.EXAMPLE`.
6. **Constrained writes.** `dev-snapshot.ps1` writes only under `_local_hold/dev_snapshots/`.
7. **No live API/LLM calls.** No script makes network calls or LLM/API invocations.
