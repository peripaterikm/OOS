## Strategic Decisions

- **Founder correction first.** Replace/amend mode is the centerpiece of v2.8, carrying forward the v2.7 policy review.
- **Artifact integrity after replacement.** Every derived artifact must be consistent after a correction.
- **Source URL traceability preserved.** No `urn:oos:*` placeholder URNs survive correction.
- **Deterministic-first preserved.** All correction logic produces deterministic output.
- **Advisory-only preserved.** No autonomous portfolio transitions.
- **Operational polish.** Windows CLI hardening and quality gate source_urls review round out the roadmap.
- **No new product layers.** v2.8 is correction workflow + hardening, not feature expansion.
- **Do not rewrite v2.6 or v2.7.** Only implement the deferred items and known operational gaps.
- **Windows-native.** PowerShell, Python venv, no WSL/Linux assumptions.
