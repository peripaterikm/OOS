# Mini-Epic 2.1 - Cluster Evidence Pack Contract

## Goal

Define a deterministic, serializable EvidencePack contract for cluster/opportunity-seed evidence so later v2.5 stages can build opportunity candidates from grounded evidence instead of loose signal lists.

## Scope

- Added `src/oos/evidence_pack.py` as a contract-only module.
- Defined `EvidencePack`, `EvidencePackItem`, `EvidencePackRiskNote`, and `EvidencePackSourceSummary`.
- Preserved evidence IDs, source URLs, source types, signal IDs, confidence values, source diversity, recurrence count, and optional price/weak-pattern/kill-warning references.
- Added helpers for deterministic pack IDs, dict serialization, dict round-trip, ordering normalization, and validation.
- Represented insufficient evidence explicitly through `created_from="insufficient_evidence"` plus required risk notes.

## Out Of Scope

- No evidence pack builder.
- No founder package display changes.
- No opportunity synthesis.
- No LLM prompt, provider, or dry-run behavior.
- No live collection or runtime artifact dependency.

## Acceptance Criteria

- [x] Evidence pack model exists and serializes.
- [x] Evidence IDs and source URLs are required for regular packs.
- [x] Price, weak-pattern, kill-warning, confidence, and risk fields are optional but structured.
- [x] Empty or insufficient evidence is represented explicitly.
- [x] Focused and full validation pass.
