# Mini-Epic 7.2: Regression Metrics for Opportunity Quality

## Purpose

Create deterministic regression metrics for the opportunity quality evaluation dataset v1 to establish a measurable baseline for the current pipeline behavior.

## Scope

- Load the 10-case opportunity quality evaluation dataset v1.
- Run the existing deterministic quality evaluation pipeline (`evaluate_opportunity_quality`) on each case.
- Compare actual gate outputs to expected `gate_decision` from the dataset.
- Produce structured, JSON-serializable aggregate metrics.
- Write focused tests covering the metrics model, serialization, validation, and per-case checks.

## Implementation Notes

### Metrics Model

`OpportunityQualityRegressionMetrics` frozen dataclass with schema version `opportunity_quality_regression_metrics.v1`.

**Aggregate fields:**
- `total_cases`
- `cases_by_quality_label` (sorted dict)
- `cases_by_founder_review_posture` (sorted dict)
- `expected_gate_decision_counts` (sorted dict)
- `actual_gate_decision_counts` (sorted dict)
- `gate_decision_matches` / `gate_decision_mismatches`
- `gate_match_rate`
- `false_positive_cases` / `false_positive_rate`
- `duplicate_cases` / `duplicate_rate`
- `unsupported_assumptions_count` / `unsupported_assumptions_cases`

**Per-case results (`PerCaseRegressionResult`):**
- `case_id`, `quality_label`, `founder_review_posture`
- `expected_gate_decision`, `actual_gate_decision`, `matched_expected_gate`
- `risk_notes`, `evidence_gaps`, `unsupported_assumptions`
- `traceability_ids` (evidence_ids, source_signal_ids, source_urls, gate_result_id, evidence_pack_id, opportunity_id)
- `false_positive_severity`, `sufficiency_band`

### False-Positive Labels

The following quality_labels are counted as false positives:
- `generic_false_positive`
- `vendor_promo_false_positive`
- `weak_noisy`
- `killed_pattern_repeat`

### Duplicate Handling

The `duplicate_signal` quality label is reported separately from false positives.

### Unsupported Assumptions

Counted from `opportunity_candidate.unsupported_assumptions` fields. All 10 cases have at least one unsupported assumption; total count = 30.

### Gate Decision Logic

No core gate, suppressor, or scoring logic was modified. The metrics module calls `evaluate_opportunity_quality()` as-is.
