## Design Decisions

1. **Append-only contract is explicit.** `_write_import_history()` loads existing history, appends, writes. No deletion or modification.
2. **Deterministic JSON.** `sort_keys=True` in `json.dumps()` and `sorted()` in all list/dict fields within `CorrectionEntry.to_dict()` ensures identical output for identical input.
3. **correction_id is deterministic.** SHA-256 of `run_id|corrected_at|mode|old_ids|new_ids` (truncated to 12 hex chars).
4. **all_replaced_decision_ids filters by mode.** Only `replace` mode entries contribute; `amend` entries excluded.
5. **Failed corrections never create history.** Validation failures return before `_write_import_history()` is called.
