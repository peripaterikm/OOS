# 4.2 Evidence Sufficiency Scoring

## Goal

Add deterministic scoring that evaluates how well an `OpportunityCandidate` is supported by evidence, separately from whether the opportunity sounds attractive.

## Scope

- Added `src/oos/evidence_sufficiency_scoring.py`.
- Added `EvidenceSufficiencyScore` with explicit dimension scores and score bands.
- Added fixture-only coverage in `tests/test_evidence_sufficiency_scoring.py`.
- Added narrow quality-gate inclusion of the sufficiency score summary.

## Scoring Dimensions

- `pain_evidence_strength`
- `workaround_evidence_strength`
- `buyer_clarity`
- `willingness_to_pay_evidence`
- `recurrence_strength`
- `source_diversity_strength`
- `traceability_strength`
- `risk_penalty`
- `ambiguity_penalty`

## Behavior

- Produces `strong`, `adequate`, `weak`, or `insufficient` score bands.
- Preserves evidence IDs, source signal IDs, and source URLs.
- Records positive factors, missing evidence, and risk factors.
- Penalizes vendor/generic/source-quality, insufficient-evidence, duplicate, and needs-more-evidence risks.
- Keeps missing price/WTP as missing evidence; it does not invent willingness-to-pay.
- Keeps recurrence and source diversity as support signals that cannot override missing pain or traceability.

## Out Of Scope

- No live LLM provider execution.
- No `provider.complete()` call.
- No live collection.
- No internet/API calls.
- No founder decision replacement.
- No auto-promotion.
- No false-positive opportunity suppressor implementation; that remains item 4.3.

## Acceptance Criteria

- [x] Each score dimension is explicit and serializable.
- [x] Missing price evidence does not create fake willingness-to-pay.
- [x] Source diversity is measured from evidence, not inferred.
- [x] Risk/ambiguity lowers confidence or gate outcome.
- [x] Full validation passes.
