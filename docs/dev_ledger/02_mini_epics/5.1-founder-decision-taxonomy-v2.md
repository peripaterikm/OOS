# 5.1 Founder Decision Taxonomy V2

## Goal

Create a structured founder decision taxonomy so founder review outcomes become consistent, machine-readable feedback instead of informal notes.

## Scope

- Added `src/oos/founder_decision_taxonomy.py`.
- Added `FounderDecisionV2`, `FounderDecisionReason`, `FounderDecisionTaxonomy`, and `FounderDecisionValidationResult`.
- Added deterministic helpers for creating, validating, serializing, round-tripping, and summarizing founder decisions.
- Added fixture-only coverage in `tests/test_founder_decision_taxonomy.py`.

## Supported Decisions

- `promote`
- `park`
- `kill`
- `revisit_later`
- `needs_more_evidence`

## Behavior

- Reasons are constrained by selected decision.
- `kill` requires a kill-category reason such as `no_buyer`, `too_generic`, or `vendor_promo_false_positive`.
- `needs_more_evidence` requires a requested evidence type such as `need_price_evidence` or `need_customer_voice`.
- Decisions preserve linked evidence IDs, source signal IDs, and source URLs where available.
- Decision IDs are deterministic unless explicitly provided.
- Decisions are record-only and do not auto-promote.

## Out Of Scope

- No decision-to-signal feedback mapping; that remains item 5.2.
- No founder preference profile; that remains item 5.3.
- No scoring changes.
- No live collection.
- No internet/API calls.
- No live LLM/API calls.
- No `provider.complete()` call.

## Acceptance Criteria

- [x] Decision taxonomy model exists and serializes.
- [x] All five decision values are supported.
- [x] Kill decisions require a structured kill reason.
- [x] Needs-more-evidence decisions preserve requested evidence type.
- [x] Full validation passes.
