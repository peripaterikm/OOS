- `ai_cfo_smb` is the only active default topic.
- `insurance_israel` exists only as an `inactive_future` stub.
- `life_management_system` exists only as an `inactive_future` stub.
- Active topics referencing undefined source IDs are rejected.

## Query Planning Rules

- Default cap: `10` query plans per source x topic.
- Default max results per plan: `25`.
- Query-kind priority: `pain_query`, `workaround_query`, `buying_intent_query`, `competitor_weakness_query`, `trend_trigger_query`.
- Dedup key: `source_id x topic_id x query_kind x normalized query_text`.
- Query plan IDs are deterministic SHA-256-derived IDs from the dedup key.
- Disabled, review-gated, unsupported, not-topic-allowed, and inactive-topic inputs produce no plans.
- `live_network_enabled` is always `false` for v2.3 planning.
