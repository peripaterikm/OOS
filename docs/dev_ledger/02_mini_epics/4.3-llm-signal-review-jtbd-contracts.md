- Roadmap: `docs/roadmaps/OOS_roadmap_v2_4_signal_quality_and_ai_layers_checklist.md`

- Commit scope: local only; no push/PR/merge/tag/release.

## Goal

Define evidence-bound contracts for future LLM-assisted signal review and JTBD extraction without enabling real LLM execution.

## In Scope

- `LLMSignalReviewInput`, `EvidenceForReview`, `LLMSignalReviewOutput`, `JTBDStatement`, and `LLMSignalReviewContract` data contracts.
- Deterministic signal-review and JTBD prompt builders using `LLMMessage`.
- Prompt-safety integration that builds a safe `LLMRequest` or fails closed before any provider call.
- Strict JSON parsing and validation for future structured LLM outputs.
- Deterministic mock review flow for tests/local preview only.
- `llm_signal_review` task type allowed by the local-preview LLM budget policy.

## Out Of Scope

- Real OpenAI, Claude, Anthropic, or other provider clients.
- Live LLM/API calls.
- Live discovery integration.
- Production LLM review execution.
- Implied burden detection roadmap item `5.1`.
- New dependencies or secrets.

## Contract Behavior

- Prompts are evidence-bound and instruct reviewers not to invent facts.
- Evidence IDs must be cited.
- The default review stance uses an asymmetric prior: assume weak/noisy signal unless evidence proves otherwise.
- Output validation fails on unknown cited evidence IDs, missing citations, invalid JSON, false `no_invention_confirmed`, invalid strength values, or out-of-range confidence values.

## Safety Behavior

- Safe request construction uses `prompt_safety` PII redaction and fail-closed blocking.
- Safe request construction does not call a provider.
- `external_calls_made` remains `False` during contract construction and validation.

## Acceptance

- Focused LLM signal review contract tests pass.
- Existing LLM budget, prompt safety, scoring, and semantic relevance regressions remain green.
- Full repository validation is recorded in the run report.
