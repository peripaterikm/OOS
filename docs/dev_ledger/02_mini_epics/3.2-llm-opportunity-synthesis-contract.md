# 3.2 LLM Opportunity Synthesis Contract

## Goal

Define a future-only LLM contract for opportunity synthesis from an `EvidencePack` and deterministic `OpportunityCandidate` baseline.

## Scope

- Added `src/oos/llm_opportunity_synthesis_contract.py`.
- Added prompt and request builders for `opportunity_synthesis`.
- Added response schema validation and evidence-bound response checks.
- Added the conservative local-preview budget role to `src/oos/llm_contracts.py`.
- Added fixture-only contract tests in `tests/test_llm_opportunity_synthesis_contract.py`.

## Contract Rules

- The LLM receives an `EvidencePack`, not a vague prompt.
- The deterministic baseline candidate is included as context and control.
- Every substantive claim must cite evidence IDs.
- Source signal IDs and source URLs must be preserved.
- Buyer, price, market size, product, strategy, and urgency must not be invented.
- Unsupported assumptions must remain explicit.
- Weak, generic, vendor-like, duplicated, or insufficient evidence must stay low-confidence.
- Output is advisory only and cannot make founder decisions.

## Out Of Scope

- No live provider integration.
- No `provider.complete()` execution.
- No runtime artifacts.
- No founder package integration.
- No opportunity dry-run; that belongs to item 3.3.

## Acceptance Criteria

- [x] Contract model/prompt exists.
- [x] Prompt requires EvidencePack context.
- [x] Prompt forbids invented buyer/price/market/product/strategy claims.
- [x] Prompt requires evidence IDs and unsupported assumption fields.
- [x] No live provider call is made by default.
- [x] Full validation passes.
