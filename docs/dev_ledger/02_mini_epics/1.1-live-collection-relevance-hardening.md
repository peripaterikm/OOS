# Mini-Epic 1.1 - Live Collection Relevance Hardening

## Goal

Improve live HN/GitHub/RSS signal quality before adding LLM, embedding, or customer-voice layers.

## Scope

- Add HTML/entity cleanup for normalized evidence text.
- Add UTF-8 decode hardening for live JSON/XML fetch paths.
- Add `ai_cfo_smb` deterministic topic relevance gating.
- Add marketing/generated-content downgrades.
- Make CandidateSignal confidence more discriminative.
- Replace broad `ai_cfo_smb` queries with finance-specific templates.
- Make RSS live collection skip non-URL query text safely.

- Customer Voice Query Generator.
- Embeddings or semantic relevance providers.

## Behavior

- Cleaner unescapes HTML entities, strips simple HTML tags, and softens obvious Markdown markers while preserving raw title/body and source URL.
- Classifier uses `ai_cfo_smb` finance anchors so generic `small business` or `spreadsheet` text does not become high-confidence pain by itself.
- Marketing/generated markers such as content calendars, product pitches, DCO campaigns, and template epics are downgraded to noise or low-confidence review.
- CandidateSignal scoring now combines classification confidence, finance relevance, pain/workaround/buying/urgency indicators, source type, and marketing penalties.
- HN/GitHub query templates now emphasize cash flow, invoices, bookkeeping, accounting, QuickBooks, Xero, reconciliation, and receivables/payables.
- RSS live mode fetches only valid `http(s)` feed URLs and reports `rss_feed_url_missing` for search text instead of throwing `unknown url type`.
