## CollectionLimits Behavior

- Defaults keep `allow_live_network` false.
- Caps include `max_total_queries`, `max_queries_per_source`, `max_queries_per_topic`, and `max_results_per_query`.
- Optional `allowed_source_ids` and `allowed_topic_ids` restrict scheduling scope.
- Validation rejects non-positive caps and malformed allow-lists.

## CollectionScheduler Behavior

- Builds a deterministic queue from `QueryPlan` inputs.
- Orders by priority, source ID, topic ID, query kind, dedup key, and query plan ID.
- Skips live-network-enabled plans unless `allow_live_network=True`.
- Enforces total, per-source, per-topic, source allow-list, and topic allow-list limits.
- Deduplicates repeated plans by `dedup_key` before collection.
- Produces `ScheduledCollectionItem` records with stable `scheduled_order`.

## Collector Interface Behavior

- `BaseCollector` defines `supports()` and `collect()` contracts.
- `CollectionResult` validates scheduled item compatibility and evidence source/topic/query-kind preservation.
- Collectors reject unsupported scheduled items rather than silently collecting.

## FixtureCollector Behavior

- Uses preloaded fixture evidence by `dedup_key` or creates deterministic offline fixture `RawEvidence`.
- Preserves `source_url`, `topic_id`, `query_kind`, and `source_id`.
- Sets `collection_method` to `fixture_collector`.
- Never uses network, API keys, secrets, or live LLM calls.
