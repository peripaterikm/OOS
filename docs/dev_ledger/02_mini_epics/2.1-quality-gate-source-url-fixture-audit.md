## Audit Findings

### Exact Root Cause

The `missing_count=1` originates from **`founder_inbox_v2_index`**, not `quality_gate_decisions`.

The specific item is `inbox_review_822b4d010950` in the `decision_recording_commands` section
of the founder inbox v2 index. This is a **legitimate synthetic inbox item** with no evidence lineage:

- `linked_opportunity_ids`: empty
- `linked_evidence_pack_ids`: empty
- `linked_evidence_ids`: empty
- `linked_quality_gate_ids`: empty
- `linked_source_urls`: empty

The `decision_recording_commands` section is a structural artifact generated by the inbox
builder (`src/oos/founder_inbox_v2.py`) that provides CLI instructions to the founder. It
has no upstream evidence references and legitimately has no source URLs.

### Fixture Source URL Verification

All 10 evaluation fixture cases in `examples/evaluation_dataset_v2_5/opportunity_quality_cases_v1.json`
were re-audited:

| Hop | Status |
|-----|--------|
| `EvidencePack.source_urls` | **All 10 non-empty** — real `http`/`https` URLs |
| `OpportunityCandidate.source_urls` | **All 10 non-empty** — faithfully propagated from evidence packs |
| `OpportunityGateResult.source_urls` | **All 10 non-empty** — faithfully propagated from candidates |

Quality gate propagation (`src/oos/opportunity_quality_gate.py`) reads `candidate.source_urls`
at line 179 and writes to `result.source_urls` at line 225. **The propagation chain is correct.
No quality gate items have missing source URLs.**

### Placeholder URN Check

Zero `urn:oos:*` placeholder URNs exist across all scanned artifacts. Placeholder URN
detection is working correctly and was not weakened.

### Classification

| Scenario | Classification |
|----------|---------------|
| `inbox_review_822b4d010950` (decision_recording_commands) | **Synthetic item** — legitimate structural artifact with no evidence lineage |
| All 10 fixture quality gate cases | **Clean** — non-empty source URLs at every hop |
| Quality gate propagation code | **Correct** — no changes needed |

## Correction Plan (for item 2.2)

### Recommended Fix

Implement a **narrow scanner exemption** in [`src/oos/source_url_traceability.py`](../../../src/oos/source_url_traceability.py)
for synthetic founder inbox items with no evidence lineage:

1. Add a helper `_is_synthetic_inbox_item_without_lineage()` that returns `True` only when:
   - `artifact_key == "founder_inbox_v2_index"`
   - All five lineage fields are empty (`linked_opportunity_ids`, `linked_evidence_pack_ids`,
     `linked_evidence_ids`, `linked_quality_gate_ids`, `linked_source_urls`)

2. Apply the exemption in `_check_artifact_source_urls()` **before** the missing-source-url
   issue is emitted.

3. Track the exemption count separately in `SourceURLTraceabilityArtifactStatus` and
   `SourceURLTraceabilityReport`.

4. **Do not weaken placeholder URN detection.** Items with non-empty `linked_source_urls`
   are still scanned. Items with any linked ID present are not exempt.

### What Must NOT Change

- `examples/` — no fixture data changes needed
- Placeholder URN detection — not weakened
- Scoring/gating/decision semantics — unchanged

## Key Decisions

1. **Audit-only scope.** Item 2.1 performed re-audit and correction plan. No implementation changes were made.

2. **Synthetic inbox exemption over fixture modification.** Adding real source URLs to the synthetic `decision_recording_commands` item would be semantically wrong — it has no evidence lineage. The exemption approach is correct and narrow.

3. **Scanner not weakened.** The exemption is specific to items where all five lineage fields are empty. Placeholder URN detection and all real missing-source-URL checks are preserved.

4. **No quality gate changes.** The audit confirmed quality gate propagation is correct. No changes to `opportunity_quality_gate.py` are needed.
