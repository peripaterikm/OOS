## Stack Exchange Collector Behavior

- Supports only `source_id=stack_exchange` and `source_type=stack_exchange`.
- Converts fixture Stack Exchange question JSON into `RawEvidence`.
- Preserves `source_url` from question `link`.
- Preserves safe metadata including `question_id`, tags, answer count, score, view count, dates, site, and `owner_present`.
- Does not store raw `display_name`, `user_id`, or owner payloads by default.
- Live network is disabled by default and requires both collector `allow_live_network=True` and scheduled item `live_network_enabled=True`.

## RSS Collector Behavior

- Supports only `source_id=rss_feeds` and `source_type=rss_feed`.
- Parses fixture RSS 2.0 feed XML with Python standard library XML parsing.
- Converts feed items into `RawEvidence`.
- Preserves `source_url` from item `link`.
- Preserves safe metadata including `guid`, `pubDate`, feed title, categories, and `original_author_present`.
- Does not store raw author names/emails by default.
- Live network is disabled by default and requires both collector `allow_live_network=True` and scheduled item `live_network_enabled=True`.

## RawEvidence Mapping

- Stack Exchange evidence IDs are deterministic: `raw_stackexchange_question_<question_id>`.
- RSS evidence IDs are deterministic: `raw_rss_<sha256-prefix>` from GUID/link/title identity.
- Both collectors preserve `source_id`, `source_type`, `topic_id`, `query_kind`, and `source_url`.
- Both collectors use existing deterministic `content_hash` behavior over title and body.

## Privacy Rule

- Stack Exchange `author_or_context` uses `unverified public question asker`.
- RSS `author_or_context` uses `public feed item` by default.
- Raw usernames, handles, display names, user IDs, and feed author identity are not stored by default.
