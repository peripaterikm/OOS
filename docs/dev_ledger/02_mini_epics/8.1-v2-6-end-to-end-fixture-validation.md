## Goal

Prove the full v2.6 weekly loop works end-to-end with deterministic fixtures, following the pattern established by v2.5 item 7.3 (`V2_5EndToEndValidationReport`).

## Key Design Decisions

- **Temp project root only**: All artifacts written to temp directories. No `artifacts/` mutation.
- **Reuses existing functions**: `build_weekly_cycle()`, `import_founder_decisions()`, `build_weekly_cycle_status()`, `build_weekly_run_report()`, `build_weekly_dashboard_index()`.
- **Fixture decisions from inbox**: Auto-generates decisions by cycling through all 5 decision types across review items.
- **Traceability is additive**: Checks are warnings, not blockers (following v2.5 pattern).
- **Fail-resilient**: If any step fails, the report captures errors and continues to verify remaining steps where possible.
- **Deterministic**: Same fixture â†’ same run_id in two separate runs.
