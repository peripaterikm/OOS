# 3.1 Deterministic Opportunity Sketch Baseline

## Goal

Create a no-LLM baseline that turns an EvidencePack into an OpportunityCandidate sketch using only explicit evidence.

## Scope

- Added a serializable `OpportunityCandidate` model in `src/oos/opportunity_sketch.py`.
- Added deterministic build/serialize/deserialize/validate helpers.
- Preserved `evidence_pack_id`, `cluster_id`, evidence IDs, source signal IDs, and source URLs.
- Added conservative baseline extraction for problem statement, target user, current workaround, why-now, possible buyer, and product wedge.
- Marked unsupported buyer/product/price/why-now assumptions explicitly as `unknown`/unsupported.
- Propagated EvidencePack risk notes and linked price, weak-pattern, and kill-warning IDs into sketch risk notes.

## Out Of Scope

- No LLM synthesis.
- No provider calls.
- No opportunity-generation pipeline integration.
- No founder package display changes.
- No live collection or source expansion.

## Acceptance Criteria

- [x] OpportunityCandidate model exists and serializes.
- [x] Baseline sketch cites and preserves evidence IDs.
- [x] Missing buyer, price, product, and why-now support remains unknown or explicitly unsupported.
- [x] Output is deterministic across repeated runs.
- [x] Insufficient/generic/vendor-risk evidence stays conservative.
- [x] Fixture-only validation passes without live internet/API or LLM calls.
