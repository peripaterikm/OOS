## Goal

Create a documented, reproducible runbook and optional automated smoke script for executing a bounded weekly cycle with fixture signals, reviewing the output, recording decisions, and verifying the full traceability chain. This runbook serves as the controlled real-run smoke test before any live collection is attempted.

Every command is copy-paste-able PowerShell. All safety boundaries are explicit.

### 2. Automated Smoke Script (`scripts/run-controlled-smoke.ps1`)

A Windows-native PowerShell script that:
- Creates a temporary project root using `[System.IO.Path]::GetTempPath()`
- Copies `src/`, `examples/`, and `config/` into the temp root
- Builds fixture input from the v2.5 evaluation dataset
- Executes the full deterministic pipeline via Python/CLI:
  1. `run-weekly-cycle-v2`
  2. Verifies `founder_inbox_v2.md` and `founder_inbox_v2_index.json`
  3. Auto-generates fixture decisions from inbox index
  4. `import-founder-decisions-v2`
  5. `weekly-cycle-status-v2`
  6. `build-weekly-run-report-v2`
  7. `weekly-dashboard-v2`
  8. `check_source_url_traceability` — verifies zero placeholder URNs
- Prints clear PASS/FAIL sections
- Returns exit code 0 on pass, 1 on fail
- Never writes to real repository `artifacts/`
- Never calls live APIs/LLMs
- Never runs git add/commit/push/merge/tag/reset/clean
- Never runs `gh pr create` / `gh pr merge`
- Uses `Set-StrictMode -Version Latest` and `$ErrorActionPreference = "Stop"`
- Has comment-based help

## Design decisions

1. **Temp-only execution.** The smoke script copies source code into a temp directory and runs the entire pipeline there. This guarantees zero contamination of the real `artifacts/` directory.

2. **Direct Python import for traceability.** The source URL traceability check uses `python -c` with a direct `check_source_url_traceability()` import rather than a separate CLI command, since no CLI subcommand exists for this function.

3. **Auto-generated fixture decisions.** The smoke script reads the inbox index, filters to items with `linked_source_urls` (non-synthetic), and generates PROMOTE decisions for them. This avoids requiring a pre-made decisions file.

4. **No Remove-Item for cleanup.** The script leaves the temp directory for OS lifecycle cleanup, printing the path for manual removal if desired. This avoids any risk of accidental deletion.
