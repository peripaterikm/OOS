# 1.4 Candidate Signal Dedup Before Founder Package

## Goal

Prevent duplicate candidate signals from appearing in founder review packages or inflating weak pattern aggregation.

## Scope

- Add a deterministic `CandidateSignal` dedup helper.
- Collapse duplicates by exact evidence ID, signal ID, or normalized source URL.
- Preserve raw evidence and raw candidate artifacts.
- Attach duplicate metadata to canonical representatives.
- Use canonical signals for founder-facing ranked output and weak pattern aggregation.

## Implementation

- Added `src/oos/candidate_signal_dedup.py`.
- Integrated canonical candidates into weekly discovery founder packages.
- Added `candidate_signal_dedup.json` discovery artifact and summary counters.
- Updated weak signal aggregation to dedup exact duplicates before cluster thresholds.

## Acceptance Criteria

- [x] Duplicate candidates collapse in founder package display.
- [x] Canonical candidate preserves duplicate evidence IDs.
- [x] Dedup does not remove distinct pains from the same source.
- [x] Weak pattern aggregation uses canonical signals.
- [x] Output ordering is deterministic.
- [x] Full validation passes.

## Out Of Scope

- No fuzzy semantic dedup.
- No embeddings or vector search.
- No deletion or rewriting of raw evidence artifacts.
- No live collection, internet/API, or LLM calls.
