# Mini-Epic 5.2 - Decision-to-Signal Feedback Mapping

## Goal

Map structured `FounderDecisionV2` records back to opportunity, evidence-pack, source-signal, evidence, and source-URL lineage so founder review becomes deterministic feedback memory.

## Scope

- Add `src/oos/founder_feedback_mapping.py`.
- Add deterministic mapping from founder decisions to feedback tags, signal impact, and recommended future handling.
- Preserve opportunity ID, evidence pack ID, optional cluster ID, evidence IDs, source signal IDs, and source URLs.
- Keep feedback advisory/record-only; do not mutate scoring.
- Avoid ML-training claims, live LLM calls, live internet calls, or live collection.

## Implemented Contract

- `FounderFeedbackTarget`
- `FounderFeedbackSignalImpact`
- `FounderFeedbackMapping`
- `FounderFeedbackMappingResult`
- `map_founder_decision_to_feedback(...)`
- Serialization, validation, deterministic ID generation, and deterministic summaries.

## Mapping Rules

- `promote` maps to positive feedback and promoted-pattern tags.
- `park` maps to neutral or needs-more-evidence feedback with missing-evidence handling.
- `kill` maps to negative or suppress-pattern feedback for generic/vendor/no-pain reasons.
- `revisit_later` maps to revisit-pattern feedback.
- `needs_more_evidence` maps to needs-more-evidence feedback and required evidence handling.

## Acceptance Criteria

- [x] Founder decisions map to opportunity IDs.
- [x] Founder decisions map to cluster IDs when available.
- [x] Founder decisions map to signal/evidence IDs.
- [x] Mapping is deterministic and serializable.
- [x] Validation coverage exists for mapping rules and traceability.

## Out Of Scope

- Founder preference profile generation.
- Scoring changes or automatic boosting/suppression.
- Live LLM/API calls.
- Live internet/API calls.
- Live collection.
