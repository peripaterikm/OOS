# Mini Epic: 3.1 — CLI and Status/Report Integration for Correction State

**Roadmap**: v2.8 — Founder Decision Correction & Operational Polish
**Item**: 3.1
**Status**: complete
**Phase**: B (implementation)
**Dependencies**: 1.3 (safe replace/amend), 2.1 (import history/audit trail)

## Goal

Ensure `weekly-cycle-status-v2`, run reports, and dashboard reflect replaced/amended decisions clearly so the founder can see correction state at a glance.

## Scope Implemented

### weekly_cycle_status.py
- Added `corrected_decision_count`, `replaced_decision_ids`, `amended_decision_ids` alias fields to `WeeklyCycleStatus`
- Added `correction_entries` list with per-entry details (correction_id, corrected_at, correction_mode, old/new decision IDs, replaced_review_item_ids)
- Added `[CORRECTED]` indicator in Run Identity section when `corrected_decision_count > 0`
- Added "Decision Corrections" Markdown section (section 11) with:
  - Corrected decision count
  - Replaced/amended decision IDs
  - Per-correction details
  - Clear "NONE" state when no import_history.json exists
- Moved "Artifact Paths" to section 12

### weekly_run_reports.py
- Added per-correction details section to `run_report.md` when import history exists
- Added "Corrections" column to dashboard Markdown table
- Added `[CORRECTED]` tag in dashboard table rows for runs with corrections
- Missing `import_history.json` handled cleanly (shows "No corrections recorded")

### Exit Code Behavior
- Corrections alone do NOT cause non-zero exit codes
- Exit code 0: valid run (including valid corrections)
- Exit code 1: issues detected (not caused by corrections)
- Exit code 2: invalid state (not caused by corrections)
