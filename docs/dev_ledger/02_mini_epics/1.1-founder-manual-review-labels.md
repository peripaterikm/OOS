# 1.1 Founder Manual Review Labels

## Goal

Create structured founder-review ground truth for all 18 candidate signals from the first real open-source signal run.

## Scope

- Add tracked label artifacts under `examples/first_real_open_source_signal_run_v1/`.
- Preserve first-run candidate signal order, including duplicate evidence IDs.
- Include `review_id`, `evidence_id`, `signal_id`, `source_type`, `source_url`, original classification/type/confidence, founder label, secondary labels, quality score, rationale, recommended action, issue tags, notes, and duplicate group IDs where applicable.
- Add fixture-only tests for allowed labels/actions, required IDs, key useful/vendor/price/duplicate cases, and issue tags.

## What Was Intentionally Not Implemented

- No suppressor or scoring changes.
- No candidate dedup implementation.
- No price extractor hardening.
- No mojibake cleanup changes.
- No source-code feature implementation.
- No additional live collection.
- No live internet/API or live LLM/API calls.

- Founder label artifact exists and serializes.
- All seven label values are supported.
- Labels preserve candidate/evidence/source traceability.
- Duplicate, vendor-promo, and price-false-positive labels are represented.
- First-run review fixture loads without live network calls.
- Full validation passes.
