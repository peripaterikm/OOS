# 8.1 Full AI meaning loop verification

## Goal

Verify the Roadmap v2.2 AI meaning loop end-to-end without live LLM/API calls and with traceability preserved across stages.

## Context

Roadmap v2.2 now has deterministic/provider-boundary implementations for dataset loading, deduplication, signal understanding, semantic clustering, contradiction detection, opportunity framing, quality gating, pattern-guided ideation, ideation comparison, anti-pattern checks, isolated council critique, founder review packaging, and founder AI-stage ratings. Item 8.1 checks that those pieces compose into one inspectable loop.

## Inputs

- `examples/evaluation_dataset_v1/`
- Existing provider/stub AI meaning modules under `src/oos/`
- FounderReviewPackage v2 writer and `record-founder-review` workflow
- Founder AI-stage rating and weekly summary support

## Outputs

- `tests/test_full_ai_meaning_loop_verification.py`
- End-to-end deterministic verification covering canonical signals, AI metadata, traceability, founder decision recording, AI quality ratings, and no-live-provider safeguards
- Roadmap state advanced to item 8.2 after green validation

## Key decisions

- Kept 8.1 as a verification layer instead of wiring new runtime behavior into `run-signal-batch`.
- Used existing static providers and contracts rather than adding new AI stages.
- Verified duplicate preservation and canonical recurrence separately so duplicate signals do not inflate semantic clustering.
- Kept founder decision authority explicit by recording a review decision only through the existing review-id workflow.

## Alternatives rejected

- Live LLM verification: rejected by Roadmap v2.2 constraints.
- New orchestration module: rejected because the current item is verification, not runtime redesign.
- Broad prose assertions: rejected in favor of structural checks for IDs, links, metadata, and artifact presence.

## Validation

- Focused: `$env:PYTHONPATH='src'; .\.venv\Scripts\python.exe -m unittest tests.test_full_ai_meaning_loop_verification -v`
- Full discovery: `$env:PYTHONPATH='src'; .\.venv\Scripts\python.exe -m unittest discover -s tests -p "test_*.py" -v`
- Project validation: `.\scripts\oos-validate.ps1`
- Final verification: `.\scripts\verify.ps1`

All required validation passed before roadmap advancement.

## Known limitations

- The test uses provider/stub payloads only; no live model quality is measured in v2.2.
- The end-to-end verification is intentionally structural and deterministic, not a benchmark.
- Dataset v1 remains synthetic and explicitly labeled.
