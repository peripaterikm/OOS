# Mini-Epic 8.1 - v2.4 End-to-End Validation

## Roadmap Item

- Roadmap: `docs/roadmaps/OOS_roadmap_v2_4_signal_quality_and_ai_layers_checklist.md`
- Item: `8.1 v2.4 end-to-end fixture and live-smoke validation`
- Branch: `feat/8-x-v2-4-final-validation`
- Commit scope: local only; no push/PR/merge/tag/release.

## Goal

Prove that the v2.4 signal quality system works from fixture and bounded live evidence through founder package artifacts.

## In Scope

- Run an end-to-end fixture discovery pass.
- Run bounded HN live smoke.
- Run bounded GitHub live smoke.
- Run bounded mixed HN/GitHub live smoke.
- Validate live smoke artifacts with the existing artifact-only smoke validator.
- Record quality observations for top signals, false positives, ambiguity, founder package quality sections, and runtime artifact handling.
- Run full local validation after smoke work.

## Out Of Scope

- Roadmap item `8.2`.
- Live LLM/API calls.
- Any LLM provider invocation or `provider.complete()` call.
- New source collectors.
- Pipeline redesign or scoring redesign.
- New dependencies.
- Unit tests that depend on live network.

## Live Call Boundaries

- Live internet/API calls are allowed only for the explicit 8.1 smoke validation runs.
- Live collector runs use small query/result caps.
- Unit tests remain fixture-only and must not make live network calls.
- Live smoke failures or warnings are recorded as observed; success is not faked.

## Acceptance Evidence

- Fixture run writes standard discovery, price-signal, founder package, and meaning-loop dry-run artifacts.
- HN, GitHub, and mixed live smokes complete through existing collector mode.
- Artifact-only live smoke validation has no failed checks in the final report, with one GitHub warning recorded.
- Top-3 results include finance pain evidence in fixture and mixed live outputs.
- Marketing/install/generic false positives do not dominate top results in the final smoke report.
- `needs_human_review` appears in live GitHub and mixed outputs for ambiguity.
- Runtime artifacts remain under ignored `artifacts/`.
