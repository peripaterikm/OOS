# Mini-Epic 2.2 - Evidence Pack Builder

## Goal

Build deterministic `EvidencePack` objects from existing OOS signal artifacts so later opportunity synthesis can work from grounded evidence instead of loose candidate signal lists.

## Scope

- Added `src/oos/evidence_pack_builder.py`.
- Built evidence packs from in-memory `CandidateSignal`, `PriceSignal`, `WeakPatternCandidate`, and kill archive warning artifacts.
- Added discovery-artifact loading for existing `candidate_signals.json`, `price_signals.json`, `weak_pattern_candidates.json`, and `kill_archive_warnings.json`.
- Linked price signals by exact `evidence_id`.
- Linked kill archive warnings by exact `evidence_id` or `signal_id`.
- Preserved evidence IDs, source URLs, source types, signal IDs, topic IDs, confidence values, source diversity, recurrence count, and risk notes.
- Added `evidence_packs.json` artifact writing in the existing weekly discovery pipeline.

## Out Of Scope

- No founder package display changes; that remains item `2.3`.
- No opportunity synthesis.
- No LLM prompt, provider, or dry-run behavior.
- No live collection or new collector behavior.
- No invented buyer, product, market size, or opportunity details.

## Acceptance Criteria

- [x] Builder reads existing discovery artifacts.
- [x] Evidence IDs/source URLs are preserved.
- [x] Price signals attach only when evidence-bound.
- [x] Kill archive warnings attach without auto-kill.
- [x] Builder output is deterministic.
- [x] Full validation passes.
