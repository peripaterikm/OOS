# 5.2 Ideation Mode Comparison

## Goal

Compare ideation modes honestly with operationalized gates, weighted metrics, thresholds, and explainable mode recommendations.

## Context

Roadmap 5.1 made pattern-guided ideation the primary meaningful ideation layer, while heuristic ideation remained baseline/fallback/control. Item 5.2 adds a deterministic comparison layer before Phase 6 anti-pattern and council critique work.

## Inputs

- Active roadmap item `5.2`
- Existing `IdeaVariant` heuristic artifacts
- Existing `PatternGuidedIdea` artifacts
- Roadmap metric table and preliminary thresholds
- Existing opportunity and signal traceability IDs

## Outputs

- `src/oos/ideation_mode_comparison.py`
- `tests/test_ideation_mode_comparison.py`
- `tests/test_ideation_mode_comparison_acceptance.py`
- README documentation for gates, metrics, thresholds, penalties, and recommendation logic

## Key Decisions

- Keep comparison standalone and not wired into `run-signal-batch`.
- Treat schema validity and traceability as gates, not scoring bonuses.
- Use the roadmap weighted formula directly.
- Keep genericness detection deterministic and intentionally simple until Phase 6.
- Preserve heuristic mode as baseline/control rather than primary intelligence.

## Alternatives Rejected

- No live LLM judgment or API call was added.
- No full anti-pattern library was implemented before Phase 6.
- No orchestrator/runtime integration was added.
- No founder-final decision flow was replaced.

## Validation

- Focused comparison tests
- Acceptance tests for no live calls, no premature orchestrator wiring, and roadmap state
- Full unittest discovery
- `scripts/oos-validate.ps1`
- `scripts/verify.ps1`

## Known Limitations

- Criterion scores are deterministic inputs or conservative defaults, not calibrated founder ratings yet.
- Genericness penalty is a small phrase-based check, not the Phase 6 anti-pattern system.
- Diversity summary is lightweight and intended for comparison, not final council selection.

## Next Step

Implement `6.1` deterministic anti-pattern checks.
