# Mini-Epic 6.2 - Next Best Founder Actions

## Goal

Create a deterministic next-best-founder-actions layer that converts the weekly opportunity review package into a prioritized list of advisory founder action recommendations.

## Scope

- Added `src/oos/next_best_founder_actions.py`.
- Defined `FounderAction` dataclass with: `action_id`, `action_type`, `title`, `rationale`, `priority`, linked IDs (`section_ids`, `item_ids`, `decision_ids`, `opportunity_ids`, `evidence_ids`, `pack_ids`), `suggested_next_step`, and `advisory_only`.
- Defined `ALLOWED_ACTION_TYPES`: `review_promote_candidate`, `collect_more_evidence`, `interview_customer`, `validate_price_signal`, `revisit_parked_opportunity`, `consider_kill_candidate`, `review_preference_warning`, `run_customer_voice_queries`, `address_evidence_gap`, `review_undecided_opportunity`.
- Implemented `build_next_best_founder_actions(weekly_review_package: dict)` — deterministic conversion from a `WeeklyOpportunityReviewPackage` dict (as produced by `weekly_review_package_to_json`) into a prioritized, flattened list of `FounderAction` items.
- Three priority bands: high (1), normal (2), low (3) — derived from section urgency.
- Ordering: deterministic by (priority asc, action_type, action_id).
- Action type and title derived from section_id and category of each `WeeklyReviewSectionItem`.
- Every action carries `advisory_only=True`.
- Stable action IDs via SHA-256 hash.
- JSON serialization (`next_best_actions_to_json`) and Markdown rendering (`render_next_best_actions_markdown`).
- Handles empty weekly review packages gracefully (returns `[]`).

## Out Of Scope

- No autonomous portfolio decisions.
- No live LLM/API calls.
- No live internet/API calls.
- No live collection.
- No CLI integration (deferred to future item).
- No portfolio transitions — actions are advisory only.
- No modifications to `weekly_opportunity_review.py` or other existing modules.

## Acceptance Criteria

- [x] Action recommendation model exists and serializes.
- [x] Actions are deterministic from gate/decision/evidence state.
- [x] Actions never bypass founder approval.
- [x] Suggested customer interviews include evidence rationale.
