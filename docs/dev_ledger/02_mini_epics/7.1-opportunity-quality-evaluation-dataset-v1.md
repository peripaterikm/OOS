# Mini-Epic 7.1 - Opportunity Quality Evaluation Dataset v1

## Goal

Create a labeled, deterministic, fixture-based dataset for evaluating opportunity quality gate decisions across diverse case types.

## Context

This dataset provides the first systematic evaluation of the OOS opportunity quality pipeline (quality gate, evidence sufficiency scoring, false positive suppression). Each case includes a synthetic evidence pack, an opportunity candidate, expected labels, and a human-readable rationale. The dataset is designed for offline, deterministic validation — no live APIs, no LLM calls, no internet.

## Scope

- Created 10 synthetic cases covering all required quality label types.
- Created a Python loader module at [`src/oos/evaluation_dataset.py`](src/oos/evaluation_dataset.py) with `load_opportunity_quality_cases_v1()`.
- Placed the canonical dataset at [`examples/evaluation_dataset_v2_5/opportunity_quality_cases_v1.json`](examples/evaluation_dataset_v2_5/opportunity_quality_cases_v1.json), following the existing project convention where evaluation datasets live under `examples/`.
- Created comprehensive validation tests at [`tests/test_opportunity_quality_evaluation_dataset.py`](tests/test_opportunity_quality_evaluation_dataset.py).

### Dataset Structure

Each case contains:
- `case_id` — unique identifier.
- `title` — human-readable case description.
- `synthetic_data` — always `true`.
- `input_artifacts` — dict with `evidence_pack` and `opportunity_candidate`.
- `expected` — dict with `quality_label`, `founder_review_posture`, `gate_decision`, `evidence_gaps`, `risk_notes`, `sufficiency_band`, `false_positive_severity`.
- `rationale` — human-readable explanation of expected outcome.

### Loader

The loader validates:
- Every case has a non-empty `case_id`, `title`, and `rationale`.
- Every case has `synthetic_data = true`.
- Every case has `expected.quality_label`, `expected.founder_review_posture`, and `expected.gate_decision`.

## Out Of Scope

- No live APIs, LLM calls, or internet/network calls.
- No modification to core opportunity/portfolio/decision logic.
- No database or persistent storage.
- No UI.
- No live collection or live signals.

## Known Limitations

- Dataset is synthetic. Real-world edge cases may produce different quality gate behavior.
- Only 10 cases cover the defined label taxonomy. Additional cases may be warranted as the system evolves.
- No mojibake-specific case (the existing mojibake cleanup is tested separately in 1.5).
- No price false-positive-specific case (tested separately in 1.3).
