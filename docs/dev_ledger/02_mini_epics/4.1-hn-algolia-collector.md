# 4.1 Hacker News Algolia Collector

## Goal

Implement the first real-source collector adapter for Hacker News Algolia while preserving fixture-first, offline-safe Roadmap v2.3 source intelligence rules.

## Scope

- Add `HNAlgoliaCollector` conforming to the `BaseCollector` contract.
- Add deterministic HN Algolia hit-to-`RawEvidence` mapping.
- Add fixture-first collection behavior with live network disabled by default.
- Add a standard-library live path gated by both collector config and scheduled item `live_network_enabled`.
- Add focused tests for source compatibility, mapping, traceability, privacy, max results, malformed hits, and the Source Registry -> Query Planner -> Scheduler -> HN collector fixture flow.

## HN Collector Behavior

- Supports only `source_id=hacker_news_algolia` and `source_type=hacker_news_algolia` by default.
- Rejects unsupported scheduled collection items.
- Uses fixture payloads when provided.
- Returns empty evidence safely when live networking is disabled and no fixture payload is provided.
- Supports only `search` and `search_by_date` endpoints for the optional future live path.

## Fixture-First Behavior

- Focused tests use in-memory HN Algolia fixture payloads.
- Empty fixture responses return an empty evidence list.
- Malformed or incomplete hits without `objectID` are skipped.
- Recoverable hits use deterministic title/body fallbacks.

## Live-Network-Disabled-By-Default Behavior

- `allow_live_network` defaults to `False`.
- The collector requires both `allow_live_network=True` and `scheduled_item.live_network_enabled=True` before using the standard-library live path.
- Tests assert no live fetch is attempted when live networking is disabled.
- No internet/API calls were made during validation.

## RawEvidence Mapping

- `evidence_id`: deterministic `raw_hn_<objectID>`.
- `source_url`: `https://news.ycombinator.com/item?id=<objectID>`.
- `title`: `title`, then `story_title`, then `HN item <objectID>`.
- `body`: `story_text`, then `comment_text`, then `url`, then `story_url`, then title.
- `source_id`, `source_type`, `topic_id`, and `query_kind` are preserved from the scheduled item.
- `content_hash` uses existing deterministic `RawEvidence` title/body hashing.
- `raw_metadata` preserves safe source metadata such as `objectID`, timestamps, points, comments, tags, original URL, author presence, query plan ID, and dedup key.

## Privacy / Author Context

- `author_or_context` is always role/context-only: `unverified public commenter`.
- Raw HN usernames are not stored in `author_or_context`.
- Raw HN usernames are not stored in `raw_metadata`; only `author_present` is retained.
