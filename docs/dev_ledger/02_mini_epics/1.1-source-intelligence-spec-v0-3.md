## Source Policy Decisions

- HN Algolia is a low-friction Phase B source with fixture/offline-first tests and live network disabled by default unless an explicit flag/config enables it.
- GitHub Issues is a low-friction Phase B source with fixture/offline-first tests, no committed tokens/secrets, and live network disabled by default unless an explicit flag/config enables it.
- Stack Exchange is a Phase B source but requires `requires_registered_app_key: true` for production/high-volume use; tests must not require a key.
- RSS/regulator/changelog feeds are Phase B sources limited to public feed content and no scraping outside feed content.
- Trustpilot/Capterra are later access-review candidates and are not guaranteed free Phase B sources.
- G2 is disabled by default, requires commercial review, and is marked `access_realistic_for_solo_founder: false`.
- Reddit is Phase D/later only, requires commercial review, and must not receive a v2.3 collector.
- LinkedIn automation is out of scope without explicit official API/legal approval.
- GDELT is experimental / Phase D because of high noise risk.
- `RawEvidence.author_or_context` stores role/context by default, not username/handle.
- `ai_cfo_smb` is the only first active topic profile.
- v2.3 feedback produces `suggested_priority_updates` only; founder approval is required before application.
- HN/GitHub ambiguous evidence defaults to `needs_human_review`, not `noise`.
- Evidence cleaner v2.3 excludes boilerplate removal.
- Every scoring dimension must define `measurement_method`.
- Traceability tests must prove `weekly_discovery_package -> candidate signal -> raw evidence -> source_url`.
