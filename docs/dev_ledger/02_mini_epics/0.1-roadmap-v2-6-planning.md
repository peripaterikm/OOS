# Roadmap v2.6 Planning

## Goal

Create the official Roadmap v2.6 checklist for Real Weekly Loop / Operationalization after the completed v2.5 system built evidence packs, opportunity candidates, quality gates, founder decisions, feedback mappings, preference profiles, weekly review packages, next-best actions, parking lot logic, evaluation datasets, and regression metrics.

## Context

Roadmap v2.5 is complete (`24 / 24`, tag `v2.5`). The post-review follow-ups were merged on `fix/v2-5-codex-review-followups`. v2.5 built all the components needed for a weekly operating loop but did not connect them into a single running pipeline. v2.6 operationalizes what v2.5 built.

### What v2.5 Delivered (Available for v2.6)

- **Evidence packs**: `EvidencePack` model, `EvidencePackBuilder`, display in founder packages
- **Opportunity candidates**: deterministic baseline synthesis, LLM synthesis contract (disabled), offline dry-run
- **Quality gates**: pass/park/reject decisions, evidence sufficiency scoring, false-positive suppression
- **Founder decisions**: `FounderDecisionV2` taxonomy with 5 decision types
- **Feedback mappings**: `FounderFeedbackMapping` from decisions to opportunity/signal/evidence lineage
- **Preference profiles**: `FounderPreferenceProfile` with preferred patterns, kill reasons, scoring hints, warnings
- **Weekly review package**: `WeeklyOpportunityReviewPackage` aggregating all of the above
- **Next-best actions**: `FounderAction` model, 10 action types, priority bands, advisory-only
- **Parking lot**: `ParkingLotRecord`, `RevisitMatch`, three-tier deterministic matching
- **Evaluation dataset**: 10 synthetic labeled cases, loader with validation
- **Regression metrics**: `OpportunityQualityRegressionMetrics`, per-case and aggregate metrics
- **End-to-end fixture validation**: `V2_5EndToEndValidationReport`, 5-stage traceability, 37 tests

### What v2.5 Did NOT Deliver (v2.6 Must Add)

- A unified function that runs the full pipeline in one pass
- A well-defined run artifact contract (what files exist, where, what schema)
- A founder inbox built from the new weekly review package (current inbox uses older format)
- A decision import mechanism that feeds decisions back into artifacts
- A rich weekly cycle status command
- Cross-run run reports and dashboard
- End-to-end validation of the full v2.6 loop

## Strategic Decisions

- **Operationalize, don't rewrite.** v2.6 connects existing v2.5 modules; it does not rebuild them.
- **Deterministic pipeline first.** The weekly cycle must complete fully without live LLM/API calls.
- **Founder-controlled decisions.** All promote/park/kill remain human decisions.
- **Markdown-first, JSON-backed.** Every output has both formats.
- **Traceability preserved.** Every item links back through evidence to source.
- **Replay-safe.** Deterministic `run_id` from input content hash.
- **Fixture-validated.** End-to-end fixture test serves as regression gate.
- **Windows-native.** PowerShell, Python venv, no WSL/Linux assumptions.
