# Mini Epic 2.1 — E2E Source URL Traceability Validation

**Roadmap:** v2.7 item 2.1
**Date:** 2026-05-08
**Status:** complete

## Goal

Add a dedicated source URL traceability verification stage to the existing v2.6 end-to-end fixture validation that asserts every artifact in the chain carries real source URLs and no placeholder URNs survive the full pipeline.

## Scope

- Extend `V2_6EndToEndValidationReport` with source URL traceability fields:
  - `source_url_traceability_validation_passed`
  - `source_url_placeholder_count`
  - `source_url_missing_count`
  - `source_url_traceability_issue_count`
  - `source_url_chain_checks`
- Add `_check_source_url_chain()` function that verifies source URL propagation across:
  1. evidence_packs → founder_inbox_v2_index
  2. founder_inbox_v2_index → founder_decisions_v2
  3. founder_decisions_v2 → founder_feedback_mappings
  4. founder_feedback_mappings → target.source_urls
- Integrate `check_source_url_traceability()` from `src/oos/source_url_traceability.py` as step s11 in the E2E validation runner.
- Add 12 focused tests covering report fields, step inclusion, zero placeholder URNs, chain verification, and safety flags.
- Keep the source URL traceability step advisory-only (always reports "passed" for pipeline status; findings are in report fields).

### Design decisions

1. **Advisory-only step.** s11 always reports `"passed"` for pipeline step status. Source URL traceability findings (missing URLs, placeholder counts, chain checks) are captured in dedicated report fields. This preserves the advisory contract from the source URL traceability contract (v1.1).

2. **Chain verification is intersection-based.** Rather than exact equality, the chain checks verify that each downstream artifact's source URL set *intersects* the upstream set. This is because:
   - Downstream artifacts may carry additional URLs from other sources.
   - The key question is traceability, not completeness.

3. **Quality gate with empty source_urls is a pre-existing gap.** The fixture `quality_gate_decisions.json` may have items with empty `source_urls`. This is a pre-existing pipeline gap, not introduced by v2.7 item 2.1. The scan detects it and reports it, but does not block the pipeline.

4. **No changes to source URL propagation.** This item verifies the chain; it does not modify how URLs propagate through the pipeline.
