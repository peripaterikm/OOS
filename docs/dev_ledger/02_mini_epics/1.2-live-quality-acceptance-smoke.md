# Mini-Epic - Roadmap v2.4 Item 1.2 Live Quality Acceptance Smoke

## Goal

Formalize the live quality acceptance smoke gate for already-generated Source Intelligence discovery runs.

## Scope

- Add a deterministic artifact-only live quality smoke checker.
- Add an opt-in CLI report command.
- Validate existing discovery run artifacts without running collectors.
- Check duplicate top source URLs, mojibake, install/tutorial leakage, GitHub/mixed noise, user-pain-like top signals, and RSS controlled skip behavior.
- Record manual live smoke outcomes and known limitations.

## Out of Scope

- Roadmap v2.4 item 2.1 Customer Voice Query Generator.
- LLM providers, embeddings, Reddit collector, or semantic review layers.
- Live internet/API calls during tests or validation.
- Collector behavior changes.

## Smoke Checker Behavior

- Reads `discovery_run_summary.json`, `founder_discovery_package.json`, and optional `meaning_loop_dry_run.json` from existing run directories.
- Produces per-run `LiveQualitySmokeReport` records and an aggregate JSON/Markdown report.
- Does not invoke collectors, APIs, network calls, or LLMs.

## Acceptance Checks

- Live run integrity: `collection_mode=live_collectors`, `live_network_enabled=true`, and meaning-loop adapted count alignment when present.
- Duplicate top `source_type + source_url` detection.
- Mojibake marker detection in top summaries.
- Install/tutorial marker failure for top five signals.
- Generic consulting/marketing copy warning, not hard failure.
- GitHub/mixed runs require `noise_count > 0`.
- Top-three user-pain-like count below two is a warning.
- RSS missing feed URL passes only when represented as controlled `rss_feed_url_missing` skip without collector failure.

## Known Limitations

- Generic finance/consulting copy can still rank too high until semantic/LLM review layers are added.
- User-pain-like matching is deterministic and heuristic.
