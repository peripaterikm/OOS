# 1.6 Second Controlled Live Run Comparison

## Goal

Run a second bounded HN + GitHub live source pass after v2.5 Block 1 hardening and compare output quality against the first real open-source signal run.

## Scope

- Reused the existing `run-discovery-weekly` pipeline with `--use-collectors` and `--allow-live-network`.
- Limited live collection to `hacker_news_algolia` and `github_issues`.
- Kept volume bounded to 4 scheduled queries, 2 queries per source, and 5 results per query.
- Compared raw evidence, candidate signals, price signals, human-review usage, dedup, vendor-promo suppression, and founder-facing mojibake.
- Recorded runtime artifacts under the existing ignored `artifacts/discovery_runs/` convention.

## Out Of Scope

- No new collectors.
- No source-code feature implementation.
- No live LLM/API calls.
- No Reddit, Facebook, LinkedIn, scraping-heavy sources, or paid APIs.
- No runtime artifacts were promoted into tracked source files.

## Why This Matters

Block 1 hardening was intended to make the same HN + GitHub source family more useful before v2.5 moves into evidence packs and opportunity formation. This comparison gives a concrete before/after checkpoint using the first run baseline and the second live artifacts.

## Acceptance Criteria

- Bounded HN + GitHub live run executed with a deterministic run id.
- Runtime artifacts generated under `artifacts/discovery_runs/second_controlled_open_source_signal_run_v1/`.
- First-run vs second-run counts summarized.
- Vendor-promo, duplicate, price false-positive, mojibake, and human-review behavior assessed honestly.
- Safe validation passed after runtime artifact handling.
- Roadmap moved to `2.1 Cluster evidence pack contract` with completed `6 / 24` and remaining `18 / 24`.

## Result

Completed locally on `feat/v2-5-block-1-first-run-quality-hardening`. The run showed clear improvements in GitHub vendor-promo suppression, duplicate collapse, price false-positive handling, and mojibake cleanup. The main warning is that exact-source dedup currently over-collapses repeated HN source URLs into a small canonical founder-facing list, so v2.5 evidence packs should preserve useful suppressed duplicate context carefully.
