## Scoring Components

Scoring v2 records:

- `topic_relevance_score`
- `pain_strength_score`
- `workaround_score`
- `buying_intent_score`
- `urgency_score`
- `source_quality_weight`
- `customer_voice_match_bonus`
- `anti_marketing_penalty`
- `signal_type_multiplier`
- `classification_confidence_factor`
- cap flags and final score
- concise explanation strings

## Formula

The current mode is `signal_scoring_v2_embeddings_disabled`.

Base weights sum to 1.0:

- topic relevance: 0.25
- pain strength: 0.25
- workaround: 0.15
- buying intent: 0.10
- urgency: 0.10
- source quality: 0.10
- customer voice match bonus: 0.05

The model then applies soft signal-type multipliers, classification confidence factor, bounded synergy boosts, anti-marketing penalty, low-relevance caps, needs-human-review cap, and noise cap.

## CandidateSignal Integration

- `CandidateSignal.confidence` now comes from Scoring model v2 `final_score`.
- `CandidateSignal.scoring_model_version` is set to `signal_scoring_v2_embeddings_disabled`.
- `CandidateSignal.scoring_breakdown` stores the component scores and explanation.
- `extraction_mode` is now `rule_based_v2`.

## Customer Voice Bonus

`customer_voice_query` inputs receive a small bonus, with optional persona/query-id explanation metadata when available. The bonus cannot override weak topic relevance or anti-marketing caps.

## Anti-Marketing Penalty

Existing deterministic marketing/install/tutorial markers feed into the scoring breakdown as `anti_marketing_penalty`, with visible caps for strong marketing/generated content.
