## Pipeline stages

1. Load signals from input file (JSON/JSONL)
2. Build evidence packs (from evaluation-dataset-style embedded packs)
3. Build opportunity candidates via `build_opportunity_sketch_from_evidence_pack()`
4. Run quality gates via `evaluate_opportunity_quality()`
5. Build founder decisions (empty — founder hasn't decided)
6. Build feedback mappings (empty)
7. Build preference profile (empty — no decisions)
8. Load prior parking lot records if `existing_artifacts_dir` provided
9. Run revisit matching if prior records exist
10. Build parking lot records from decisions
11. Build weekly opportunity review package
12. Build next-best founder actions
13. Write all 14 artifacts to `artifacts/weekly_runs/{run_id}/`
14. Write manifest.json

## Key design decisions

- **No CandidateSignal construction**: The builder does NOT construct intermediate `CandidateSignal` objects. Evidence packs are parsed directly from evaluation-dataset-style input dicts. This avoids the complex `CandidateSignal` constructor signature and keeps the pipeline focused on evidence packs → opportunity candidates.
- **Placeholder artifacts**: Founder inbox v2 and run report artifacts are written as deterministic placeholders with explicit notes that they will be fully implemented in later v2.6 items (4.1, 7.1).
- **Empty founder decisions**: The builder does not simulate founder decisions. `founder_decisions_v2.json` is written as an empty array with an explanatory note. Founder decisions are imported separately via item 5.1.
