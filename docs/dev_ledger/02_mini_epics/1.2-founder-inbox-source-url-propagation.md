# Mini Epic 1.2 — Founder Inbox Source URL Propagation

**Roadmap:** v2.7 item 1.2
**Status:** complete
**Depends on:** 1.1 (source URL traceability contract)
**Implemented:** 2026-05-08

## Goal

Add `linked_source_urls` field to `FounderInboxReviewItem` and populate it from upstream evidence packs, opportunity candidates, and quality gate results during inbox generation.

## Known Gap (from v2.6)

`FounderInboxReviewItem` had `linked_evidence_ids`, `linked_opportunity_ids`, `linked_source_artifact_ids`, but no `linked_source_urls`. The inbox index JSON is the machine-readable record that the founder decision import reads from. Without source URLs in the inbox, the decision import cannot propagate real URLs.

## Implementation

### Source changes

- **`src/oos/founder_inbox_v2.py`:**
  - Added `linked_source_urls: list[str]` field to `FounderInboxReviewItem` with default `[]`.
  - Updated `to_dict()` to include `linked_source_urls`.
  - Added `_resolve_source_urls()` helper that collects source URLs from upstream artifacts:
    1. Evidence packs matching `linked_evidence_pack_ids`
    2. Opportunity candidates matching `linked_opportunity_ids`
    3. Quality gate results matching `linked_quality_gate_ids`
    4. Evidence packs containing `linked_evidence_ids` in their items
  - Returns deduplicated, deterministically sorted, non-empty http/https URLs only. No placeholder URNs.
  - Added `_post_process_source_urls()` that resolves and populates `linked_source_urls` for every review item in place.
  - Integrated post-processing into `build_founder_inbox_v2()` after all sections are built, before traceability summary.
  - Updated `_build_traceability_summary()` to include `unique_source_urls`, `items_with_source_urls`, `items_without_source_urls`.
  - Updated `render_founder_inbox_v2_markdown()` to display `Source URLs` per item and source URL counts in the Traceability Appendix.

### Test changes

- **`tests/test_founder_inbox_v2.py`:**
  - Added `FounderInboxSourceURLPropagationTests` class with 14 focused tests covering:
    1. `linked_source_urls` serialization in `to_dict()`
    2. Empty upstream results in `linked_source_urls=[]`
    3. Evidence pack source URLs propagate to top opportunities
    4. Evidence pack source URLs propagate to needs-more-evidence
    5. Opportunity candidate source URLs propagate
    6. Quality gate source URLs propagate
    7. Duplicate URL deduplication (deterministic, sorted)
    8. Markdown includes source URL traceability
    9. Weekly cycle builder writes `linked_source_urls` in index JSON
    10. Source URL traceability scanner finds URLs in inbox index (no placeholder flags)
    11. No `urn:oos:*` placeholder introduced
    12. Backward compatibility: existing inbox tests pass with new field defaulting to `[]`
    13. Integration: build with packs+gates carries real URLs
    14. Traceability appendix has URL counts

### Scope boundaries preserved

- Did NOT modify `founder_decision_import.py`.
- Did NOT remove `urn:oos:founder_import:placeholder` (deferred to 1.3).
- Did NOT change `FounderDecisionV2` or `FounderFeedbackMapping` behavior.
- Did NOT change portfolio behavior.
- Did NOT implement decision re-import.
- Did NOT modify scoring/gating logic.
- Did NOT call live APIs/LLMs.
- Did NOT write real `artifacts/` during tests.
