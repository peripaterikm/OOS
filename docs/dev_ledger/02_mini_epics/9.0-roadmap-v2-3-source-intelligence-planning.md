## Key decisions

- Roadmap v2.3 starts at item `1.1` with `0 / 16` completed and `16 / 16` remaining.
- Reddit is Phase D/later only and requires commercial review.
- G2 is disabled and unrealistic for a solo founder unless access/cost review proves otherwise.
- Stack Exchange is Phase B but requires registered app key policy for production/high-volume use.
- Feedback loop produces `suggested_priority_updates` only; founder approval is required before application.
- Privacy defaults to storing role/context in `author_or_context`, not usernames or handles.
- `ai_cfo_smb` is the first active topic; other named topics are inactive future stubs only.
- HN/GitHub ambiguous evidence defaults to `needs_human_review`, not `noise`.
- Evidence cleaner v2.3 avoids boilerplate removal.
- Every scoring dimension must expose `measurement_method`.
- Traceability acceptance tests must prove weekly package to source URL lineage.

## Alternatives rejected

- Leaving v0.1/v0.2 as loose roadmap files: rejected to keep `docs/roadmaps/` focused on active source-of-truth roadmaps.
- Deleting v0.1/v0.2: rejected because they are useful planning history.
- Implementing collectors during planning: rejected because this checkpoint is roadmap/spec only.
- Treating G2/Reddit as near-term sources: rejected by access/commercial policy.
- Automatic feedback application in v2.3: rejected; deferred to v2.4+.
