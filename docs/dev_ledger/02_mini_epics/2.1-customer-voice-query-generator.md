# Mini-Epic - Roadmap v2.4 Item 2.1 Customer Voice Query Generator

## Goal

Add a deterministic Customer Voice Query Generator framework so Source Intelligence can search in realistic customer/operator language, not only founder or expert language.

- Add proposed-by-default approval state fields.
- Add a disabled-by-default future LLM prompt contract.
- Add inactive future topic stubs.
- Add a CLI preview command that writes JSON and Markdown artifacts.

## Persona-Based Design

The framework uses `topic -> persona set -> customer voice queries -> source-aware query planning -> future yield tracking`.

The active topic is `ai_cfo_smb`, but the persona set is not owner-only. It includes:

- `smb_owner`
- `bookkeeper`
- `accountant`
- `fractional_cfo`
- `finance_manager`
- `operations_manager`
- `freelancer_solo_operator`

`freelancer_solo_operator` is included because freelancer/solo finance sits between personal finance and SMB finance and can reveal strong AI-CFO-adjacent pain.

## Query Behavior

- Query IDs are deterministic.
- Query ordering is stable.
- `query_kind` is `customer_voice_query`.
- `generation_method` is `deterministic_seed`.
- `approval_state` defaults to `proposed`.
- Queries include `query_intent`, `expected_source_fit`, `rationale`, and tags.
- Queries are not activated for live collection by this item.

## Future Topic Stubs

Inactive stubs were added for:

- `freelancer_solo_finance`
- `immigrant_finance_israel`
