# 4.1 Post-Synthesis Quality Gate

## Goal

Add a deterministic post-synthesis gate that reviews evidence-bound `OpportunityCandidate` outputs after deterministic or offline/mock synthesis.

## Scope

- Added `src/oos/opportunity_quality_gate.py`.
- Added serializable `OpportunityGateReason` and `OpportunityGateResult` structures.
- Added `evaluate_opportunity_quality(...)` for pass/park/reject decisions.
- Preserved evidence IDs, source signal IDs, and source URLs in gate output.
- Added fixture-only tests in `tests/test_opportunity_quality_gate.py`.

## Behavior

- Returns deterministic `pass`, `park`, or `reject` decisions.
- Explains decisions with structured reasons, blocking issues, missing evidence, and recommended next action.
- Keeps founder decision authority explicit with `founder_decision_required = true`.
- Keeps `auto_promote = false` for every valid result.
- Rejects missing traceability, generic/unknown problems, and vendor/generic false-positive risk.
- Parks plausible but incomplete opportunities that need buyer, workaround, product wedge, source diversity, price, or other evidence.

## Out Of Scope

- No live LLM provider execution.
- No `provider.complete()` call.
- No live collection.
- No internet/API calls.
- No founder package integration.
- No discovery runtime artifact writer.
- No evidence sufficiency dimension scoring beyond the narrow 4.1 gate rules.

## Acceptance Criteria

- [x] Gate returns pass/park/reject deterministically.
- [x] Gate never auto-promotes.
- [x] Gate preserves founder-decision authority.
- [x] Unsupported assumptions push toward park/reject.
- [x] Full validation passes.
