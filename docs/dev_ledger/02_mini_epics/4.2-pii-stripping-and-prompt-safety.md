# 4.2 PII Stripping and Prompt Safety

## Goal

Add a deterministic privacy and prompt-safety layer that future LLM review tasks can use before any model request is built.

## Scope

- Added `src/oos/prompt_safety.py`.
- Added PII finding and redaction result models.
- Added prompt safety policy and report models.
- Added deterministic PII detection and redaction for common sensitive patterns.
- Added fail-closed prompt safety evaluation.
- Added safe `LLMMessage` and `LLMRequest` builders.
- Added a generic prompt safety envelope with asymmetric prior and evidence-citation requirements.
- Added focused tests in `tests/test_prompt_safety_pii.py`.

## Out Of Scope

- No real LLM provider calls.
- No OpenAI, Anthropic, Claude, or local model clients.
- No LLM Signal Review or JTBD extraction contracts from item 4.3.
- No PII stripping connected to live discovery execution by default.
- No dependencies, secrets, internet calls, or live LLM calls.

## PII Redaction Behavior

- Detects and redacts email addresses, phone-like numbers, URLs, IPv4 addresses, valid credit-card-like numbers, Israeli ID-like numbers with checksum, IBAN-like strings, bank-account-like long digit sequences, obvious API tokens, and private-key blocks.
- Uses deterministic stdlib regex/checksum logic only.
- Preserves the original text in the redaction result while returning a redacted text copy for prompt construction.
- Does not claim complete PII coverage and intentionally avoids aggressive person-name detection.

## Prompt Safety Behavior

- `default_prompt_safety_policy()` redacts PII, requires redaction before LLM use, blocks secrets/private keys/cards, and fails closed.
- `local_preview_prompt_safety_policy()` keeps deterministic local-preview behavior while still redacting PII and blocking high-risk secrets/cards/private keys.
- `evaluate_prompt_safety()` returns a structured report with findings, warnings, block reasons, and `external_calls_made=False`.
- `build_safe_llm_request()` returns a redacted request when safe and `None` when blocked.

## Prompt Envelope Behavior

- `build_prompt_safety_envelope_message()` returns a generic system message for future LLM tasks.
- The envelope requires redacted evidence, asymmetric prior, no invention, evidence ID citation, and `evidence_cited = true` only when evidence is cited.
