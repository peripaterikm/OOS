# Mini-Epic: 1.3 Safe Replace/Amend in Founder Decision Import

**Roadmap:** v2.8 item 1.3
**Status:** complete
**Branch:** feat/v2-8-decision-correction-block-1
**Date:** 2026-05-09

## Goal

Implement safe, explicit founder decision correction modes in the founder decision import flow:
1. **Replace mode** (`--replace-review-items`): surgical replacement of explicitly listed review items with full derived artifact rebuild.
2. **Amend-notes-only mode** (`--amend-notes-only`): notes/reason category amendment with decision value preserved.

Default reject-on-reimport behavior remains unchanged.

## Technical Decisions

1. **Import history kept inside `founder_decision_import.py`** — cleaner than separate module; models are small
2. **Atomic writes via write-then-rename** — `_write_json_atomic()` writes to `.tmp`, validates parseability, then renames
3. **No replace-all implemented** — per contract Section 4.4 recommendation, deferred to v2.9+
4. **amend-notes-only validates decision value matches** — cannot change decision type; uses separate `_import_amend_notes_only()` path
5. **`_map_existing_decisions_to_rid()`** helper maps review_item_ids to existing FounderDecisionV2 objects via inbox opportunity_id matching
6. **Checksums computed before and after writes** for `import_history.json` entries

## Risks / Follow-up

1. **Amend mode does not rebuild derived artifacts** — by design (notes-only). If downstream artifacts need update, use replace mode.
2. **No undo/rollback** — v2.9+ candidate
3. **Run report/dashboard regeneration** — handled by existing CLI commands (`build-weekly-run-report-v2`, `weekly-dashboard-v2`); not automated in correction path
4. **Checksums are SHA-256[:16]** — truncated for size; full digests could be used if needed
