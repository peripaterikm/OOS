# Mini-Epic 6.3 - Parking Lot / Revisit Logic

## Goal

Keep parked opportunities visible and revisit them when new matching evidence appears, using deterministic matching only.

## Scope

- Added `src/oos/parking_lot.py` with two advisory-only models:
  - `ParkingLotRecord` — a parked or revisit-later opportunity snapshot with record_id, source_decision_id, linked opportunity/artifact IDs, title/summary, reason, pattern_keys, status (parked/revisit_later), and advisory_only=True.
  - `RevisitMatch` — a deterministic match between new evidence and a parked record with match_id, parking_lot_record_id, matched evidence/opportunity IDs, match_reason, matched_pattern_keys, confidence (low/medium/high), suggested_founder_action, and advisory_only=True.
- Implemented `build_parking_lot_records(decisions)` — builds `ParkingLotRecord` items from PARK and REVISIT_LATER founder decisions. PROMOTE and KILL decisions are excluded. Malformed decisions are safely skipped.
- Implemented `match_revisit_candidates(parking_lot_records, new_evidence)` — deterministic matching between new evidence/signals and parked records using three-tier strategy:
  1. Exact pattern_key intersection → confidence "high"
  2. Conservative normalized token overlap (≥2 significant tokens) → confidence "medium"
  3. Conservative normalized substring match (≥2 significant tokens) → confidence "low"
- No embeddings, ML, LLM, network calls, or live APIs.
- No autonomous portfolio transitions. Founder remains final decision-maker.
- All IDs are deterministic via SHA-256 hashing.
- JSON serialization and Markdown rendering helpers included.

### Weekly Review Integration

- Modified `src/oos/weekly_opportunity_review.py`:
  - Added optional `parking_lot_records` and `revisit_matches` parameters to `build_weekly_opportunity_review_package()`.
  - Updated `_build_revisit_queue()` to accept and surface `RevisitMatch` items alongside existing REVISIT_LATER decisions.
  - When no parking lot data is provided, behavior is unchanged.
  - Revisit matches appear with confidence-based urgency (high/normal/low) and link to matched evidence/opportunity IDs.

## Out Of Scope

- No autonomous portfolio transitions.
- No embeddings, ML, or LLM-based matching.
- No live network/API calls.
- No database or persistent storage (in-memory deterministic only).
- No UI.
