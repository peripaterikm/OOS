# Mini-Epic - Roadmap v2.4 Item 2.2 Customer Voice Query Planner Integration

## Goal

Make deterministic Customer Voice queries planner-compatible while preserving the product rule that customer-language queries do not automatically affect live collection.

## Scope

- Add approval-gated conversion from `CustomerVoiceQuery` to `QueryPlan`.
- Preserve `query_kind = customer_voice_query`.
- Respect `expected_source_fit`, source filters, collector availability, and topic/source policy.
- Add deterministic preview CLI output for inspectable query plans.
- Keep default query planning and live collection unchanged.

## Out Of Scope

- No Roadmap v2.4 item 3.1 scoring work.
- No LLM provider calls.
- No embeddings.
- No Reddit collector implementation.
- No live collector execution from customer voice queries.
- No dependencies or secrets.

## Planner Behavior

- Proposed, rejected, and retired customer voice queries are excluded from planner output.
- Approved customer voice queries can produce deterministic `QueryPlan` records.
- Query plans preserve source, topic, query text, `customer_voice_query` kind, bounded max results, and live-network disabled state.
- `raw_metadata` records `persona_id`, `customer_voice_query_id`, `query_intent`, `generation_method`, and `approval_state`.

## Approval Gate

- Customer voice planner integration is disabled by default because generated queries start as `approval_state=proposed`.
- The preview CLI emits zero plans unless `--approve-generated-preview-queries` is explicitly provided.
- The preview approval flag approves generated deterministic copies only for preview output and does not mutate source registry or saved query definitions.

## Source Gating

- `expected_source_fit` controls eligible source types.
- `--source-type` filters source types/source IDs before plans are emitted.
- Sources without collectors, commercial-review-blocked sources, and inactive/future topic sources remain excluded.
- Reddit remains blocked because the Reddit collector is unavailable.

## CLI Preview

Command:

```powershell
.\.venv\Scripts\python.exe -m oos.cli preview-customer-voice-query-plans `
  --topic ai_cfo_smb `
  --source-type github_issues `
  --max-queries-per-persona 2 `
  --max-total-queries 10 `
  --approve-generated-preview-queries `
  --output artifacts/customer_voice_queries/ai_cfo_smb_customer_voice_query_plans.json `
  --output-md artifacts/customer_voice_queries/ai_cfo_smb_customer_voice_query_plans.md
```

The command writes deterministic JSON/Markdown and reports `collectors_run=false`, `live_network_calls=false`, and `llm_calls=false`.
