# Mini-Epic 3.1: CLI Command for Unified Weekly Cycle

## Status: Complete

## Scope

Add a new CLI subcommand `run-weekly-cycle-v2` to expose the unified weekly cycle builder (`build_weekly_cycle()`) through the existing `oos.cli` entry point.

## Implementation

### Files Changed

- `src/oos/cli.py` — Added `run-weekly-cycle-v2` subparser and handler
- `tests/test_weekly_cycle_cli_v2.py` — 20 focused CLI tests

### CLI Command

```
python -m oos.cli run-weekly-cycle-v2 --project-root . --input-file <path>
```

### Arguments

| Argument | Type | Required | Description |
|----------|------|----------|-------------|
| `--project-root` | Path | No (default cwd) | OOS project root directory |
| `--input-file` | Path | **Yes** | JSONL, JSON array, or evaluation dataset file |
| `--run-id` | str | No | Explicit run ID (auto-generated if omitted) |
| `--prior-artifacts-dir` | Path | No | Prior weekly run dir for parking lot revisit |

### Behavior

1. Calls `build_weekly_cycle()`, passing all arguments through
2. Prints structured summary to stdout:
   - `run_id`, `run_dir`, `manifest_path`
   - `artifact_count`, `validation_passed`
   - `warnings_count`, `errors_count`
   - Pipeline summary: input signals, evidence packs, opportunity candidates, quality gates, next-best actions, parking lot
   - Advisory flags: `advisory_only`, `no_live_api`, `no_live_llm`
3. Exit codes:
   - `0` — validation_passed=True
   - `1` — validation_passed=False
   - `2` — ValueError during build (invalid args)
