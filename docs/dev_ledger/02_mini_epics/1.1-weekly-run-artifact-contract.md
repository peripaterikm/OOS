# Mini-Epic 1.1 - Weekly Run Artifact Contract

## Goal

Define the exact set of artifacts that one v2.6 weekly cycle run must produce. This is the contract ‚Äî every consumer (founder inbox, status command, run reports, import, replays) reads from these well-known paths.

## Scope

- Added `src/oos/weekly_run_manifest.py`.
- Defined `WeeklyRunManifest` dataclass with `run_id`, `created_at`, `schema_version`, `artifact_paths`, `artifact_schema_versions`, `empty_states`, `input_file`, `input_signal_count`, and advisory/no-live safety flags.
- Defined 14 canonical artifact paths under `artifacts/weekly_runs/{run_id}/`:
  - `manifest.json` ‚Äî `WeeklyRunManifest`
  - `evidence_packs.json` ‚Äî EvidencePack[]
  - `opportunity_candidates.json` ‚Äî OpportunityCandidate[]
  - `quality_gate_decisions.json` ‚Äî QualityGateDecision[]
  - `founder_decisions_v2.json` ‚Äî FounderDecisionV2[]
  - `founder_feedback_mappings.json` ‚Äî FounderFeedbackMapping[]
  - `founder_preference_profile.json` ‚Äî FounderPreferenceProfile
  - `weekly_opportunity_review.json` ‚Äî WeeklyOpportunityReviewPackage
  - `next_best_actions.json` ‚Äî FounderAction[]
  - `parking_lot_records.json` ‚Äî ParkingLotRecord[]
  - `run_report.json` ‚Äî WeeklyRunReport
  - `founder_inbox_v2_index.json` ‚Äî machine-readable inbox index
  - `founder_inbox_v2.md` ‚Äî human-readable Markdown inbox
  - `run_report.md` ‚Äî human-readable Markdown run report
- Implemented deterministic `run_id` generation: `weekly_run_{YYYY-MM-DD}_{content_hash_short}` where `content_hash = sha256(input_file_content + run_date.isoformat())[:12]`.
- Implemented `write_weekly_run_manifest()` ‚Äî validates and writes `manifest.json` to a run directory.
- Implemented `read_weekly_run_manifest()` ‚Äî reads, parses, validates, and path-traversal-guards a `manifest.json`.
- Implemented `make_default_manifest()` ‚Äî creates a fully-populated manifest with all 14 artifact paths, schema versions, and empty-state flags.
- Validation rejects: missing required fields, unknown schema versions, unknown artifact keys, path traversal (..), absolute paths, missing mandatory `manifest` entry, and false advisory/no-live flags.
- Empty-state run produces a valid manifest with all `empty_states` flags set to `true`.
- JSON roundtrip is stable: same `to_dict()` ‚Üí `json.dumps()` ‚Üí `json.loads()` ‚Üí `from_dict()` produces equal manifest.
- Deterministic key ordering in `to_dict()` via canonical key list.
