# 1.2 GitHub Vendor-Promo / SEO Suppressor

## Goal

Reduce false positives from GitHub Issues where vendor, SEO, product-listing, or promotional accounting copy was being interpreted as user pain or workaround evidence.

## Scope

- Add a deterministic vendor-promo suppressor for first-run GitHub false positives.
- Integrate suppression into evidence classification before signal extraction.
- Apply a scoring cap when promotional content is retained as `needs_human_review` for traceability.
- Surface the suppression cap in founder-package risk notes when a retained candidate is displayed.
- Protect real feature requests and organic HN pain signals from vendor-promo suppression.
