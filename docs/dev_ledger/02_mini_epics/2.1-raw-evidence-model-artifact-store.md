## Privacy Rule

`RawEvidence.author_or_context` stores role/context by default. Tested allowed examples include `SMB owner`, `developer`, `founder`, and `unverified public commenter`.

Obvious direct handle forms such as `@direct_handle`, `u/direct_handle`, and `username: direct_handle` are rejected by validation.

## Artifact Store Behavior

- `RawEvidence.id` maps to `evidence_id` for compatibility with the existing `ArtifactStore.write_model` convention.
- `MODEL_KIND[RawEvidence]` maps to `raw_evidence`.
- Roundtrip path is `artifacts/raw_evidence/<evidence_id>.json`.
- `source_url` is preserved exactly.
- `raw_metadata` survives JSON serialization/deserialization.
