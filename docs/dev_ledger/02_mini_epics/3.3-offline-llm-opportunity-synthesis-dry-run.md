# 3.3 Offline LLM Opportunity Synthesis Dry Run

## Goal

Run opportunity synthesis through a deterministic/mock offline path using the 3.2 contract and the 3.1 deterministic baseline.

## Scope

- Added `src/oos/llm_opportunity_synthesis_dry_run.py`.
- Added `OpportunitySynthesisDryRunResult`.
- Added deterministic mock response generation from `EvidencePack` and `OpportunityCandidate`.
- Added schema validation and evidence-bound response checks using the 3.2 contract.
- Added fixture-only tests in `tests/test_llm_opportunity_synthesis_dry_run.py`.

## Behavior

- Builds a deterministic `OpportunityCandidate` baseline from an `EvidencePack`.
- Builds an `opportunity_synthesis` request without executing any provider.
- Produces a deterministic mock synthesis response from explicit evidence only.
- Preserves evidence IDs, source signal IDs, and source URLs.
- Records prompt hash, prompt preview, confidence, risk notes, unsupported assumptions, and cited evidence IDs.
- Keeps weak/vendor/generic and insufficient evidence low-confidence.

## Out Of Scope

- No live LLM provider execution.
- No `provider.complete()` call.
- No live collection.
- No internet/API calls.
- No runtime artifact writer or CLI integration.
- No founder package integration.
- No quality gate implementation.

## Acceptance Criteria

- [x] Dry-run reads in-memory EvidencePack fixtures.
- [x] Dry-run uses deterministic/mock synthesis only.
- [x] Output preserves evidence IDs.
- [x] Invalid/unsupported fields fail closed or mark low confidence.
- [x] No live LLM/API calls.
- [x] Full validation passes.
