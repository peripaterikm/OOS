## Summary

Implemented the `--utf8` opt-in flag on three CLI subcommands as specified by the output mode contract (item 1.1). Created a centralized `src/oos/output_modes.py` helper providing canonical symbol mappings for both `ascii_safe` (default) and `utf8` (opt-in) modes. Updated status, run report, and dashboard Markdown renderers to accept `output_mode` parameter. ASCII-safe default behavior is preserved; CP1251/CP1252 compatibility is guaranteed.

## Audit Result: import-founder-decisions-v2

The `import-founder-decisions-v2` terminal output (lines 1094-1116 in cli.py) prints only field labels, values, and boolean flags with zero status/quality symbols. No `--utf8` flag was added. Confirmed by test `test_import_decisions_help_excludes_utf8`.

## Output Mode Helper Behavior

- `validate_output_mode(mode)` — Validates against `frozenset({"ascii_safe", "utf8"})`; raises `ValueError` for unknown modes
- `get_output_symbols(mode)` — Returns dict with keys: success, failure, warning, none, arrow, dash, corrected
- ASCII-safe symbols: `OK`, `FAIL`, `WARN`, `NONE`, `->`, `-`, `[CORRECTED]`
- UTF-8 symbols: `✓` (U+2713), `✗` (U+2717), `⚠` (U+26A0), `NONE`, `→` (U+2192), `—` (U+2014), `[CORRECTED]`

## ASCII Default Behavior

No flag → `ascii_safe` mode → all output characters are `ord(c) < 128`. Verified by 3 tests checking every character of default output.

## UTF-8 Opt-In Behavior

`--utf8` → utf8 mode → Unicode checkmarks, crosses, arrows, and em dashes replace ASCII equivalents where semantically appropriate. Verified by tests checking for specific Unicode code points.
