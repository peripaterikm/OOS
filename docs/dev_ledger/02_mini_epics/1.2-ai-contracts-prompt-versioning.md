# 1.2 AI Contracts And Prompt Versioning

## Goal

Create shared metadata, prompt versioning, input hashing, cache key, and budget foundations.

## Context

Multiple AI-shaped artifacts need comparable metadata before any live LLM integration.

## Inputs

- Roadmap v2.2 metadata requirements
- Existing ideation artifacts

## Outputs

- `src/oos/ai_contracts.py`
- Prompt identity and deterministic input hash helpers
- Budget threshold helpers for economy/standard/deep modes
- Focused tests

## Key Decisions

- Prompt identity must include explicit name and version.
- Cache keys use `input_hash + prompt_version + model_id`.
- Metadata can describe fallback/degraded operation.

## Alternatives Rejected

- Silent prompt edits under the same version.
- Adding live provider calls before metadata exists.

## Validation

- AI contract focused tests.
- Existing ideation tests.
- Full unittest discovery and `scripts/verify.ps1`.

## Known Limitations

- Budget tracking is a foundation, not full runtime accounting.

## Next Step

Reuse metadata in future AI-shaped stages.
