# Mini Epic 7.1: Run Reports and Dashboard Index

**Roadmap**: v2.6 Real Weekly Loop / Operationalization
**Item**: 7.1
**Status**: Complete
**Date**: 2026-05-07

## Goal

Add structured run reports and a cross-run dashboard index so the founder can see trends across weekly cycles, not just the current one.

## Scope

- Create `WeeklyRunReport` model with per-run aggregation.
- Create `WeeklyDashboardIndex` model with cross-run aggregation.
- Create `WeeklyDashboardRunSummary` model for per-run dashboard rows.
- Implement `build_weekly_run_report()` consuming a run directory and producing a report.
- Implement `build_weekly_dashboard_index()` scanning all runs and producing an index.
- Implement `write_weekly_run_report()` writing `run_report.json` + `run_report.md`.
- Implement `write_weekly_dashboard_index()` writing `dashboard_index.json` + `dashboard.md`.
- Implement Markdown renderers for both report and dashboard.
- Add CLI subcommands: `build-weekly-run-report-v2`, `weekly-dashboard-v2`.
- Replace placeholder run report logic in `weekly_cycle_builder.py` with real implementation.
- Write focused tests.
- Update roadmap checklist and project state.

## Key Design Decisions

- **Uses WeeklyCycleStatus as source of truth**: Does not duplicate status logic.
- **Deterministic**: Same inputs produce same outputs. `generated_at` can be pinned.
- **Read-only where possible**: Report reads from disk; CLI writes only report/dashboard artifacts.
- **Advisory-only**: All safety flags preserved. No portfolio mutations.
- **No live APIs/LLMs**: Asserted in models and tests.
