## CLI Command

```powershell
python -m oos.cli run-discovery-weekly --topic ai_cfo_smb --project-root .
```

Optional flags:

- `--run-id <id>`
- `--input-raw-evidence <path>`

## Input Behavior

- Loads local `RawEvidence` JSON from `--input-raw-evidence` when provided.
- Defaults to `examples/source_intelligence_mvp/raw_evidence_seed.json`.
- Uses fixture/local data only.
- Rejects unknown or inactive topics before writing run artifacts.

## Output Artifacts

The CLI writes under `artifacts/discovery_runs/<run_id>/`:

- `raw_evidence_index.json`
- `cleaned_evidence.json`
- `evidence_classifications.json`
- `candidate_signals.json`
- `discovery_run_summary.json`
- `discovery_run_summary.md`

## Summary Behavior

`discovery_run_summary.json` includes:

- run id and topic id
- evidence/classification/signal counts
- counts by source type
- counts by classification
- counts by signal type
- needs-human-review and noise counts
- artifact paths

`discovery_run_summary.md` includes a concise MVP CLI lite summary and a short candidate signal list. It is not the final founder package.

## Traceability Behavior

Candidate signals preserve:

- `evidence_id`
- `source_url`
- `source_id`
- `source_type`
- `topic_id`
- `query_kind`

The summary and signal artifacts preserve source URL traceability.
