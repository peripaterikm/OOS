# Roadmap v2.7 Planning

## Goal

Create the official Roadmap v2.7 checklist for Traceability Hardening & Real-Run Readiness after the completed v2.6 system delivered the real weekly operating loop with CLI, founder inbox v2, founder decision import, weekly cycle status, run reports, dashboard index, and end-to-end fixture validation.

## Context

Roadmap v2.6 is complete (`9 / 9`). v2.6 connected all v2.5 components into a running weekly operating loop. v2.7 hardens the traceability chain and founder workflow so the weekly loop is production-ready for real, repeated use.

### What v2.6 Delivered (Available for v2.7)

- **WeeklyRunManifest**: 14 artifact paths, deterministic `run_id`, read/write helpers
- **Unified weekly cycle builder**: `build_weekly_cycle()` running the full pipeline
- **CLI**: `run-weekly-cycle-v2`, `import-founder-decisions-v2`, `weekly-cycle-status-v2`, `build-weekly-run-report-v2`, `weekly-dashboard-v2`
- **Founder Inbox v2**: `FounderInboxReviewItem`, 10-section Markdown, JSON index, decision recording commands
- **Founder Decision Import**: `import_founder_decisions()`, fail-closed, idempotent, 5 decision types
- **Weekly Cycle Status**: 10-section status view, auto-discovers latest run
- **Run Reports & Dashboard**: `WeeklyRunReport`, `WeeklyDashboardIndex`, per-run and cross-run reporting
- **Fixture E2E Validation**: `V2_6EndToEndValidationReport`, 11 pipeline steps, 8-link traceability chain
- **1400 tests, 0 failures**

- No developer workflow helper scripts (git snapshot, final validation, post-merge sync, PR readiness)
- No controlled real-run smoke test runbook
- No documented founder decision re-import policy

## Strategic Decisions

- **Traceability hardening first.** Every artifact in the chain must carry real `source_urls`; placeholder URNs are a bug.
- **Real-run readiness.** The founder must be able to run a bounded weekly cycle and verify the full traceability chain.
- **Deterministic-first preserved.** All hardening maintains deterministic output; no live LLM/API calls by default.
- **Advisory-only preserved.** No autonomous portfolio transitions.
- **No new product layers.** This is hardening, not feature expansion.
- **Do not rewrite v2.6.** Only fix known gaps and add workflow support.
- **Conditional scope for re-import policy.** Phase B (implementation) only proceeds if justified by policy review.
- **Windows-native.** PowerShell, Python venv, no WSL/Linux assumptions.
