# Mini-Epic 2.2 — Quality Gate Source URL Strictness / Synthetic Inbox Exemption

**Roadmap:** v2.9 item 2.2
**Status:** Complete
**Branch:** `feat/v2-9-output-source-recovery-block-1`
**Depends on:** 2.1 (audit)

## Goal

Implement the correction plan from item 2.1 audit: add a narrow scanner exemption in `source_url_traceability.py` for synthetic founder inbox items with no evidence lineage, so fixture E2E source URL traceability can pass with `missing_count=0` while preserving placeholder URN detection and all real missing-source checks.

## Audit Findings (from 2.1)

- `missing_count=1` originates from `founder_inbox_v2_index`, not `quality_gate_decisions`.
- The affected item is `inbox_review_822b4d010950`, section `decision_recording_commands`.
- It is a legitimate synthetic inbox item with no linked IDs and no linked source URLs.
- All 10 fixture cases have non-empty source URLs at every hop. Quality gate propagation is correct.

## Scope

- Add `_is_synthetic_inbox_item_without_lineage()` helper to `src/oos/source_url_traceability.py`.
- Apply exemption before missing-source-url issue is emitted in `_check_artifact_source_urls()`.
- Add `items_exempt_synthetic_inbox` and `exempt_synthetic_inbox_count` fields to report models.
- Update docstring for `check_source_url_traceability()`.
- Add 4 focused tests in `tests/test_source_url_traceability.py`.
- Update E2E validation test expectations in `tests/test_v2_6_end_to_end_weekly_cycle_validation.py`.
- Update contract, deferral document, roadmap, and project state.

## Design Decisions

1. **All-five-empty check.** Exemption only when all five fields are empty: `linked_opportunity_ids`, `linked_evidence_pack_ids`, `linked_evidence_ids`, `linked_quality_gate_ids`, `linked_source_urls`. Any single non-empty field disqualifies the exemption.
2. **Placeholder URN detection preserved.** Items with non-empty `linked_source_urls` are still scanned for `urn:oos:*` placeholders regardless of other link fields.
3. **Narrow scope.** Only `founder_inbox_v2_index` artifact. No other artifact type gets this exemption.
4. **Counted/documented.** Exemption counts are tracked separately from insufficient-evidence exemptions.
