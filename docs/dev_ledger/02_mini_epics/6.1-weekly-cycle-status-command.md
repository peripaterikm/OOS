## Summary

Added a deterministic read-only weekly cycle status command (`weekly-cycle-status-v2`) that inspects an existing weekly run directory and reports on its completeness, artifact status, and next steps. No artifact modification, no rebuild, no import, no portfolio mutation, no live APIs/LLMs.

## Models

- **`WeeklyCycleStatus`**: Full run status with run_id, manifest_valid, artifact counts/statuses, inbox/decision/feedback/profile/parking lot/action status, warnings/errors, recommended_next_step, safety flags, validation_passed.
- **`WeeklyCycleArtifactStatus`**: Per-artifact status: exists, is_empty_state, schema_version, item_count, warnings, errors.

## CLI Command

```
python -m oos.cli weekly-cycle-status-v2 --project-root <dir> [--run-id <run_id>] [--json]
```

- `--run-id` optional — auto-discovers latest run by directory name (descending).
- `--json` optional — output JSON instead of Markdown.

## Exit Codes

- 0: run exists, all artifacts present, validation_passed
- 1: run exists but manifest valid with validation issues (missing artifacts, corrupt JSON)
- 2: invalid run_id, no runs found, or missing/invalid manifest
