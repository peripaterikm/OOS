## Goal

Validate the full v2.5 pipeline end-to-end on the opportunity quality fixture dataset, proving that all advisory layers function correctly without live APIs or autonomous decisions.

## Scope

- Load the v2.5 opportunity quality evaluation dataset (10 labeled cases).
- For each fixture case, validate evidence pack and opportunity candidate artifacts.
- Run the deterministic quality gate (`evaluate_opportunity_quality()`).
- Compute opportunity quality regression metrics (`compute_regression_metrics()`).
- Create representative founder decisions from fixture expectation posture/gate decision.
- Map founder decisions to feedback mappings (`map_founder_decision_to_feedback()`).
- Build founder preference profile (`build_founder_preference_profile()`).
- Build parking lot records and revisit matches.
- Build weekly opportunity review package (`build_weekly_opportunity_review_package()`).
- Build next-best founder actions (`build_next_best_founder_actions()`).
- Preserve traceability across evidence, opportunity, gate output, founder feedback, weekly review, and next actions.
- Advisory only: no autonomous portfolio transitions.
- Deterministic ordering and stable IDs.
- JSON-serializable validation report.
- No live APIs, no LLM calls, no internet access.

## Key Implementation Decisions

1. **Fixture-driven founder decisions**: The `_derive_founder_decision_and_reasons()` function maps fixture `expected.founder_review_posture` to a `FounderDecisionV2` and reason categories. This preserves the "advisory only" posture by never auto-promoting.

2. **Deterministic matching**: `match_revisit_candidates()` uses `new_evidence` items built from evidence pack summaries, not evidence packs directly, matching the actual function signature.

3. **Validation report model**: `V2_5EndToEndValidationReport` provides all required fields: `total_cases`, `cases_processed`, `gate_decision_counts`, `weekly_review_sections_present`, `next_best_actions_count`, `traceability_checks`, `advisory_only_checks`, `failed_cases`, `warnings`, `validation_passed`.

4. **No modifications to core modules**: All core scoring/gate/feedback logic is reused as-is. No core module modifications were required.
