# 7.2 Founder rating and evaluation dataset v1

## Goal

Let the founder rate AI output quality by stage and expand the evaluation dataset from v0 to v1.

## Context

Roadmap 7.1 made FounderReviewPackage v2 readable without artifact hunting. Roadmap 7.2 adds advisory stage-quality feedback so future prompt/model changes can be compared against founder judgment.

## Inputs

- FounderReviewPackage v2 `ai_quality` section
- Existing weekly review artifact flow
- Evaluation dataset v0 structure and loader pattern
- Roadmap-approved rating values and AI stages

## Outputs

- `src/oos/founder_ai_stage_rating.py`
- `record-ai-stage-rating` CLI command
- Weekly summary field for recent advisory AI-stage ratings
- `examples/evaluation_dataset_v1/` with 22 explicitly synthetic signals and expected notes
- Focused and acceptance tests for ratings and dataset v1

## Key decisions

- Ratings are advisory and observational only; they do not change portfolio state or replace founder decisions.
- Ratings are stored as explicit JSON artifacts under `artifacts/ai_stage_ratings/`.
- FounderReviewPackage v2 surfaces ratings through the existing `ai_quality` section discovery hook.
- Dataset v1 remains fixture-only and Windows-friendly; synthetic rows are explicitly labeled.

## Alternatives rejected

- Automatic decision-making from ratings: rejected because founder decision remains final authority.
- A broad evaluation CLI: deferred because a dataset loader is sufficient for repeatable tests in this mini-epic.
- Live LLM scoring: rejected by scope; no live API calls are allowed.

## Validation

Focused tests, full unittest discovery, `oos-validate.ps1`, and `verify.ps1` passed before roadmap advancement and local commit.

## Known limitations

- Ratings are manually recorded and do not yet aggregate into prompt/model dashboards.
- Dataset v1 uses realistic synthetic signals because no larger real fixture set is available in the repo.
- Expected notes define targets for later evaluation but do not implement clustering or scoring.
