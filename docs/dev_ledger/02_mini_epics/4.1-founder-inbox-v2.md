# Mini Epic: 4.1 — Founder Inbox v2

## Roadmap Reference

- Roadmap: `docs/roadmaps/OOS_roadmap_v2_6_real_weekly_loop_operationalization_checklist.md`
- Item: `4.1`
- Dependencies: `2.1` (unified weekly cycle builder)

## Scope

Replace the placeholder `founder_inbox_v2.md` and `founder_inbox_v2_index.json` artifacts with a real, deterministic founder-facing inbox built from the full weekly cycle pipeline outputs.

## Implementation Summary

### New Module

[`src/oos/founder_inbox_v2.py`](../../src/oos/founder_inbox_v2.py)

**Models:**
- `FounderInboxV2` — top-level inbox container with run metadata, sections, review items, traceability summary, warnings, errors, and safety flags
- `FounderInboxSection` — a named section with items and explicit empty state
- `FounderInboxReviewItem` — a single reviewable item with review_item_id, traceability links, decision options, priority, and advisory_only flag

**Builder:**
- `build_founder_inbox_v2(...)` — deterministic builder consuming weekly review package dict, founder actions dicts, quality gate results, evidence packs, opportunity candidates, revisit matches, and parking lot records. All inputs optional — missing inputs produce empty states.

**Renderer:**
- `render_founder_inbox_v2_markdown(inbox)` — produces deterministic Markdown with 10 sections + traceability appendix
- `founder_inbox_v2_to_json(inbox)` — produces JSON-serializable dict

### Integration

[`src/oos/weekly_cycle_builder.py`](../../src/oos/weekly_cycle_builder.py) now:
- Imports and calls `build_founder_inbox_v2`, `render_founder_inbox_v2_markdown`, `founder_inbox_v2_to_json`
- Writes real `founder_inbox_v2.md` and `founder_inbox_v2_index.json` instead of placeholders
- Removed old `_build_founder_inbox_v2_placeholder` and `_build_founder_inbox_v2_index_placeholder` functions
- Updated `pipeline_stage_status` to report `"founder_inbox": "completed"` instead of `"placeholder"`

### Markdown Sections (10 sections)

1. Executive Summary
2. Top Opportunities to Review
3. Promote Candidates
4. Park / Revisit Later Candidates
5. Kill / Reject Candidates
6. Needs More Evidence
7. Revisit Queue / Parking Lot
8. Suggested Next Actions
9. Preference Profile Warnings
10. Decision Recording Commands

Every section has explicit empty state.

### JSON Index

Full machine-readable companion with:
- schema_version, inbox_id, run_id, generated_at
- advisory_only, no_live_api, no_live_llm flags
- source_manifest_path, markdown_path
- sections, review_items, review_item_count
- traceability_summary, warnings, errors

### Safety Guarantees

- No autonomous decisions (advisory_only=True everywhere)
- No live API/LLM calls
- No portfolio mutations
- No auto-promote/park/kill
- No founder decision import
- Deterministic output (same inputs → same review_item_ids)
- Malformed upstream data gracefully skipped
- Missing optional inputs produce empty states, not crashes
