# Mini Epic: 6.1 — End-to-end Correction Workflow Validation

**Roadmap**: v2.8 — Founder Decision Correction & Operational Polish
**Item**: 6.1
**Status**: complete
**Phase**: B (implementation)
**Dependencies**: 1.1 (correction artifact contract), 1.2 (rebuild model), 1.3 (safe replace/amend), 2.1 (import history), 3.1 (status/report integration), 4.1 (Windows CLI hardening), 5.1 (quality gate source URLs review)

## Goal

Add deterministic end-to-end validation for the founder decision correction workflow from initial import through replace, amend, derived artifact rebuild, parking lot cleanup, import history, status/report/dashboard visibility, and source URL traceability.

## Scope Implemented

### New Module: `src/oos/v2_8_correction_workflow_validation.py`

Public API:
- `V2_8CorrectionWorkflowValidationReport` — full validation report dataclass
- `V2_8CorrectionWorkflowValidationStep` — per-step result dataclass
- `run_v2_8_correction_workflow_validation(project_root=None)` — main validation runner
- `v2_8_correction_workflow_validation_to_json(report)` — JSON serialization

### Validation Flow (14 steps: C1–C14)

1. **C1** — Build weekly cycle from `sample_signals_batch_01.json`
2. **C2** — Confirm 14 weekly run artifacts exist
3. **C3** — Confirm founder inbox has review items
4. **C4** — Generate and import initial decisions
5. **C5** — Find a PARK or REVISIT decision for replace test (fallback to any decision if none)
6. **C6** — Replace decision with non-parking decision (PROMOTE)
7. **C7** — Verify unrelated decisions preserved after replace
8. **C8** — Verify import_history.json records the replace entry
9. **C9** — Verify derived artifacts rebuilt (feedback mappings, preference profile, parking lot)
10. **C10** — Verify parking lot cleanup consistency
11. **C11** — Amend notes-only on another decision
12. **C12** — Verify amend preserves decision value
13. **C13** — Verify status/report/dashboard show correction state
14. **C14** — Verify source URL traceability: zero `urn:oos:*` placeholder URNs

### Report Fields
- `schema_version`, `generated_at`, `validation_passed`, `steps`
- `run_id`, `temp_project_root`, `initial_decision_count`
- `replace_operation_summary`, `amend_operation_summary`

- `parking_lot_cleanup_summary`, `source_url_traceability_summary`
- `warnings`, `errors`
- `advisory_only`, `no_live_api`, `no_live_llm`
