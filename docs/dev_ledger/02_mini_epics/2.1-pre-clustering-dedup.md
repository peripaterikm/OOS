# 2.1 Pre-Clustering Dedup

## Goal

Prevent duplicate or near-duplicate signals from inflating recurrence before semantic clustering.

## Context

Recurrence is cluster-level, and duplicate raw signals must stay traceable without overcounting.

## Inputs

- Signal artifacts
- Roadmap rule: near duplicate means cosine similarity `>= 0.85` on normalized signal text

## Outputs

- `src/oos/signal_dedup.py`
- Duplicate metadata fields
- Canonical signal set helper
- Focused dedup tests

## Key Decisions

- Preserve all original signals.
- Add `duplicate_group_id`, `is_duplicate`, and `canonical_signal_id`.
- Use canonical signals for downstream recurrence-sensitive stages.

## Alternatives Rejected

- Physically deleting duplicates.
- Choosing an undocumented near-duplicate algorithm.

## Validation

- Dedup/fingerprinting tests.
- Real signal batch compatibility tests.
- Full unittest discovery and `scripts/verify.ps1`.

## Known Limitations

- Dedup is deterministic preprocessing, not semantic clustering.

## Next Step

Feed canonical signals into signal understanding and clustering boundaries.
