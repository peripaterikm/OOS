# 7.1 FounderReviewPackage v2

## Goal

Create a fixed founder review package structure so the founder can review the AI meaning loop without artifact hunting.

## Context

The existing founder review flow created legacy ops files and supported `record-founder-review --review-id ...`. Roadmap 7.1 adds a richer package layout while preserving that decision workflow.

## Inputs

- Existing `FounderReviewEntry` records
- Existing source artifacts under `artifacts/`
- Legacy founder review index and inbox requirements
- Roadmap v2.2 fixed package structure

## Outputs

- `artifacts/founder_review/inbox.md`
- `artifacts/founder_review/index.json`
- `artifacts/founder_review/sections/*.md` for signals, dedup, clusters, opportunities, ideas, anti-patterns, critiques, decisions, and AI quality
- Focused and acceptance tests for v2 package structure
- README documentation for FounderReviewPackage v2

## Key decisions

- Preserve legacy `artifacts/ops/founder_review_index.json` for existing `review_id` decisions.
- Emit v2 package files in addition to legacy files rather than replacing them.
- Missing optional AI-stage artifacts are documented in section files instead of crashing generation.
- Sections link to source artifact IDs where available.

## Alternatives rejected

- Replacing the legacy ops index: rejected because it would risk breaking existing founder decision flows.
- Wiring new AI stages into the runtime package: rejected because Roadmap 7.1 is package structure only.
- Failing package generation when optional artifacts are absent: rejected because early runs may not have every AI-stage artifact yet.

## Validation

- Focused tests verify package structure, section files, index metadata, source links, review-id workflow, artifact preservation, and missing optional artifact behavior.
- Acceptance tests verify the roadmap structure, no live API calls, preserved decision workflow, and roadmap advancement.
- Final validation passed before local commit.

## Known limitations

- Some section files may contain only linked artifact IDs until later stages produce richer artifacts.
- Weekly status still points to the legacy ops inbox/index for compatibility.
