# 4.1 LLM Provider Contracts and Budgets

## Goal

Add a safe LLM infrastructure contract layer for future Roadmap v2.4 work without enabling real provider clients or live model calls.

## Scope

- Added `src/oos/llm_contracts.py` with request/response/usage/provider dataclasses and protocol.
- Added `DisabledLLMProvider` as the default fail-closed provider.
- Added `DeterministicMockLLMProvider` for local tests and previews only.
- Added budget policy and budget state models.
- Added deterministic token and zero-cost estimators.
- Added circuit breaker state/control model.
- Added focused tests in `tests/test_llm_contracts_and_budgets.py`.

## Out Of Scope

- No real OpenAI, Anthropic, Claude, local model, or OpenAI-compatible provider clients.
- No PII stripping or prompt safety envelope from item 4.2.
- No LLM Signal Review/JTBD contracts from item 4.3.
- No secrets, API keys, dependencies, internet calls, or live LLM calls.

## Provider Behavior

- `disabled` is the default provider and returns `llm_disabled` without a response.
- `deterministic_mock` is local/test-only and returns deterministic content derived from task type and request text hash.
- Unknown provider IDs raise `ValueError` deterministically.

## Budget Behavior

- `default_disabled_llm_budget_policy()` fails closed with zero calls/tokens/cost and no allowed task types.
- `default_local_preview_llm_budget_policy()` allows bounded local-preview tasks only.
- Budget checks reject disallowed tasks, exhausted calls, oversized per-call input/output, run-token overflow, cost overflow, and open circuit breakers.
- Usage recording increments counters and opens the circuit breaker when run limits are exceeded.
