# Mini Epic: 1.3 — Founder Decision Import Source URL Propagation

**Roadmap:** v2.7 Traceability Hardening & Real-Run Readiness
**Item:** 1.3 Founder Decision Import source URL propagation
**Status:** Complete
**Branch:** `feat/v2-7-traceability-block-1`
**Depends on:** 1.1 (contract), 1.2 (inbox URLs)

## Goal

Replace the `urn:oos:founder_import:placeholder` fallback in [`src/oos/founder_decision_import.py`](../../src/oos/founder_decision_import.py) with real source URLs propagated from the founder inbox v2 index, evidence packs, and opportunity candidates.

## Scope

1. Read `linked_source_urls` from `founder_inbox_v2_index.json` review items.
2. Propagate real source URLs to:
   - `FounderDecisionV2.linked_source_urls`
   - `FounderFeedbackMapping.source_urls`
   - `FounderFeedbackMapping.target.source_urls`
3. Remove the `urn:oos:founder_import:placeholder` fallback entirely.
4. Add `_resolve_import_source_urls()` helper that accepts only real http/https URLs.
5. Fail-closed if an imported decision cannot resolve required source URLs (empty `linked_source_urls` for non-exempt review items).
6. Add placeholder URN rejection to `FounderDecisionV2` validation.
7. Add placeholder URN rejection to `FounderFeedbackMapping` validation.
8. Update E2E fixture decision builder to filter synthetic inbox items and require `linked_source_urls`.
9. Update test fixtures that create mock inbox items to include `linked_source_urls`.

## Implementation Notes

### Source URL Resolution Policy

- **Primary source:** `linked_source_urls` from the inbox index review item.
- **URL filtering:** Only `http://` and `https://` URLs are accepted; `urn:oos:*` placeholders are filtered out deterministically.
- **Deduplication:** URLs are deduplicated and sorted deterministically.
- **Empty URL handling:** If no real URLs are resolved from the inbox, the import fails closed with a clear error message unless the item is explicitly exempt.
- **No placeholder generation:** `_resolve_import_source_urls()` never creates `urn:oos:*` values.

### Fail-Closed Behavior

- If `linked_source_urls` is empty for a non-exempt review item, the import fails with: `review_item_id '...': no source URLs resolved from inbox linked_source_urls...`
- No partial artifacts are written.
- All existing fail-closed behaviors preserved: duplicate `review_item_id`, unknown `review_item_id`, invalid decision values, invalid reason categories.

### Validation Hardening

- `FounderDecisionV2.validate()` now rejects `urn:oos:*` patterns in `linked_source_urls`.
- `FounderFeedbackMapping.validate()` now rejects `urn:oos:*` patterns in both `source_urls` and `target.source_urls`.
- Empty `linked_source_urls` remains a warning in taxonomy validation (not a hard error) to allow synthetic items.
