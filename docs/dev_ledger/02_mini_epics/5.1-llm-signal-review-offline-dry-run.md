# Mini-Epic 5.1 - LLM Signal Review Offline Dry-Run

## Roadmap Item

- Roadmap: `docs/roadmaps/OOS_roadmap_v2_4_signal_quality_and_ai_layers_checklist.md`
- Item: `5.1 LLM Signal Review offline dry-run`
- Branch: `feat/5-x-llm-review-offline-dry-run`
- Commit scope: local only; no push/PR/merge/tag/release.

## Goal

Run the LLM Signal Review and JTBD contracts against existing discovery artifacts in a deterministic offline dry-run without real provider calls.

## In Scope

- Load `candidate_signals.json` and `cleaned_evidence.json` from an existing discovery run or explicit file paths.
- Select CandidateSignals deterministically by confidence and ID.
- Build one evidence-bound `LLMSignalReviewInput` per selected signal.
- Build safe `LLMRequest` objects through prompt safety.
- Run `run_deterministic_mock_signal_review` only.
- Validate `LLMSignalReviewOutput` and JTBD citation contracts.
- Write deterministic JSON and Markdown reports.
- Add a guarded CLI command for offline dry-run execution.

## Out Of Scope

- Real LLM provider calls.
- Provider selection.
- Live discovery integration.
- Live internet/API calls.
- Production LLM review execution.
- Roadmap item `5.2`.
- New dependencies or secrets.

## Safety

- `llm_calls_made=false` and `external_calls_made=false` are recorded in every report.
- Blocked prompt-safety items produce no review output.
- Provider `.complete()` is never called.
- Prompt safety redaction happens before mock review execution.

## CLI

```powershell
python -m oos.cli run-llm-signal-review-dry-run `
  --project-root . `
  --discovery-run-id mvp_launch_001 `
  --review-run-id llm_review_dry_run_001 `
  --topic ai_cfo_smb `
  --max-signals 5 `
  --min-confidence 0.5
```

- Existing LLM contract, prompt safety, candidate extraction, and discovery CLI regressions remain green.
