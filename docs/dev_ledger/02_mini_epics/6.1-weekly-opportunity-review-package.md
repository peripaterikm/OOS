# Mini-Epic 6.1 - Weekly Opportunity Review Package

## Goal

Create a deterministic `WeeklyOpportunityReviewPackage` that aggregates existing founder decisions, feedback mappings, preference profiles, and opportunity candidates into a founder-facing weekly review with structured sections, clear empty states, and artifact traceability.

## Scope

- Added `src/oos/weekly_opportunity_review.py`.
- Defined `WeeklyOpportunityReviewPackage`, `WeeklyReviewSection`, and `WeeklyReviewSectionItem` dataclasses.
- Implemented `build_weekly_opportunity_review_package(...)` — deterministic aggregation from existing decision/feedback/profile/opportunity artifacts.
- Ten required sections: `top_opportunities_to_review`, `promote_candidates`, `park_candidates`, `kill_candidates`, `needs_more_evidence`, `revisit_queue`, `evidence_gaps`, `suggested_interviews_or_validation`, `suggested_next_queries`, `preference_profile_warnings`.
- Each section has a clear empty state when no input data is available.
- Every section item preserves traceability: `source_artifact_type`, `source_artifact_id`, linked decision/feedback/evidence/opportunity IDs.
- Advisory only: `advisory_only=True`, `autonomous_decisions_made=False` enforced by validation.
- Deterministic ID based on SHA-256 of sorted input artifact IDs.
- JSON serialization and Markdown rendering functions.

## Out Of Scope

- No autonomous portfolio decisions.
- No live LLM/API calls.
- No live internet/API calls.
- No live collection.
- No CLI integration (deferred to future item).
- No revisit queue logic (deferred to 6.3).

## Acceptance Criteria

- [x] Weekly review package model exists.
- [x] All required sections render deterministically.
- [x] Empty sections have clear empty states.
- [x] Package cites opportunity/evidence IDs.
- [x] Full validation passes.
