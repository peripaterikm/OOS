## Goal

Define the canonical output mode contract for OOS CLI and Markdown/terminal rendering. This is a contract/docs-first item: no implementation, no source code changes, no tests.

## Motivation

v2.8 item 4.1 (commit `e36a470`) replaced Unicode symbols with ASCII-safe alternatives in CLI output, fixing CP1251/CP1252 rendering issues. However, acceptance criterion 4.1.3 ("`--utf8` flag forces Unicode output") was marked complete but never implemented. A code audit at the start of v2.9 confirmed zero occurrences of `utf8` in `src/oos/`.

This mini-epic creates the authoritative specification that item 1.2 will implement.

## Contract Scope

The output mode contract defines:

1. **Purpose** — Why output modes exist, why ASCII-safe is default, why UTF-8 is opt-in.
2. **Output modes** — `ascii_safe` (default), `utf8` (opt-in), unknown mode fail-closed behavior.
3. **CLI policy** — Which commands get `--utf8` in item 1.2, which are excluded, audit requirements.
4. **Symbol mapping table** — Canonical ASCII-safe and UTF-8 symbol mappings.
5. **Compatibility rules** — ASCII-only guarantee, JSON artifact independence, Markdown artifact policy, output-mode-independent business logic.
6. **Testing requirements** — ASCII-safety checks, Unicode presence checks, CP1251/CP1252 compatibility, no regression, minimum 12 tests.
7. **Non-goals** — No encoding auto-detection, no rich formatting, no color, no business logic changes.
8. **Acceptance criteria for item 1.2** — Preview of what item 1.2 must satisfy.

## Key Design Decisions

| Decision | Rationale |
|----------|-----------|
| ASCII-safe is the default | Safe for all Windows terminals (CP1251/CP1252) without user configuration |
| `--utf8` is opt-in, single boolean flag | Unambiguous, idempotent, no per-symbol granularity |
| No automatic terminal encoding detection | Platform-specific, non-deterministic; deferred to v2.10+ |
| Unknown mode fails closed | Silent fallback would hide configuration errors |
| `[CORRECTED]` unchanged in both modes | Semantic label, not decorative symbol |
| JSON artifacts are mode-independent | Structure and values unaffected by output mode |
| Only 3 commands get `--utf8` in v2.9 | Minimal surface area; expansion requires justification |
