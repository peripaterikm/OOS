# Mini-Epic 5.3 - Founder Preference Profile Draft

## Goal

Create a deterministic advisory `FounderPreferenceProfile` that aggregates existing `FounderDecisionV2` and `FounderFeedbackMapping` records into structured pattern memory. The profile feeds scoring hints and founder package warnings without mutating scores, making autonomous decisions, or claiming ML training.

## Scope

- Added `src/oos/founder_preference_profile.py`.
- Defined `FounderPreferenceProfile`, `FounderPreferenceScoringHint`, and `FounderPreferencePackageWarning` dataclasses.
- Implemented `build_founder_preference_profile(decisions, feedback_mappings)` — deterministic aggregation from existing decision/feedback artifacts.
- Implemented `profile_scoring_adjustment(profile, pain_type, ...)` — advisory-only scoring hints (boost/suppress/require-evidence) clamped to `[-0.20, 0.10]`.
- Implemented `profile_founder_package_warnings(profile)` — five warning categories: `portfolio_health`, `opportunity_discovery`, `evidence_gap`, `kill_pattern_memory`, `preference_gap`.
- Extended `SignalScoringInput` and `SignalScoreBreakdown` with `founder_preference_advisory_adjustment`, `founder_preference_hint_kind`, and `founder_preference_hint_reason` fields.
- Integrated preference adjustment into `build_signal_score_breakdown` in `signal_scoring.py`.
- Added `founder_preference_profile_warnings` section to `founder_package.py` quality sections.

## Out Of Scope

- No ML training claims or behavior.
- No autonomous portfolio decisions.
- No live LLM/API calls.
- No live internet/API calls.
- No live collection.
- No scoring mutation (advisory only).

## Acceptance Criteria

- [x] Profile model exists and serializes.
- [x] Promoted/rejected patterns are deterministic summaries.
- [x] Recurring kill reasons are surfaced.
- [x] Profile creates advisory scoring/founder-package warnings only.
- [x] No ML training claims or behavior.
- [x] Focused and full validation pass.
